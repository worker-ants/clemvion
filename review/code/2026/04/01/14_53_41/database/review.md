해당 없음

변경된 코드는 프론트엔드 인증 토큰 관리(`refreshPromise` 중복 방지, `sessionRestoreInProgress` 플래그) 및 백엔드 auth 컨트롤러의 요청 검증 로직(refresh token optional화, 쿠키 path 수정)에 관한 것으로, 데이터베이스 쿼리, 스키마, 트랜잭션, 마이그레이션, 커넥션 관리 등 데이터베이스와 직접적인 관련이 없습니다.

### 요약

이번 변경사항은 클라이언트 사이드 인증 토큰 관리 개선과 백엔드 refresh 엔드포인트의 입력 검증 강화에 집중되어 있습니다. 데이터베이스 레이어에 영향을 주는 코드 변경은 포함되어 있지 않습니다.
