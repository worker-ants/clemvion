### W1 (side_effect) — 진입-시 폐기가 **정당한 리셋을 삼킨다**. **확인·수정 (내 fix 를 되돌림)**

`11_38_14` W1("차단된 부팅의 리셋이 다음 부팅으로 샌다")을 고치면서 리뷰어 원안(**BLOCKED 직전 폐기**)을 기각하고 "`applyConfig` 진입 시 일괄 폐기"를 택했다. 기각 사유는 "조기 return 마다 폐기를 추가해야 해서 새 조기 return 이 생기면 재발한다" 였다 — **그건 가정이었고, 내 대안은 실제 버그를 만들었다.**

- **재현(내 실측)**: 구 세션 존재 → boot#1 in-flight → `resetSession` → boot#1 여전히 in-flight 중 boot#2 도착 → 둘 다 허용 → **`hookPosts=0 exec=OLD OLD잔존=true`**. host 가 명시 요청한 "새 대화"가 완전히 소실되고 구 세션이 이어졌다. side_effect 의 3원 mutation 대조와 일치.
- **리뷰어의 원인 진단이 정확했다**: 진입-시 폐기가 없으면 **"먼저 소비한 쪽이 `newChat`→`teardownSession`→세대 증가로 나머지 부팅을 stale 화"** 하는 자기치유가 성립한다. 내 fix 는 그 구조를 깨고 "진입~소비 시간 구간" 기준을 도입했는데, 실제 필요 기준은 **"생존한 시도"** 였다.
- **수정**: 진입-시 폐기를 되돌리고 **BLOCKED 분기에서만** 폐기(= 리뷰어 원안). 자기치유를 보존하면서 `11_38_14` W1 도 닫힌다. 주석에 **"진입 시 일괄 폐기로 바꾸지 말 것"** 과 그 이유를 명시했다 — 다음 사람이 같은 함정에 빠지지 않도록.
