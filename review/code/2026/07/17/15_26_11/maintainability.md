대상: `codebase/channel-web-chat/src/widget/use-widget-eager-start.test.ts` — 겹친 부팅 시나리오 테스트 2건에
"전제 고정(premise-pinning)" 단언 추가 + 리드 코멘트 정확도 정정. **프로덕션 코드 0줄 변경**(diff 대조 확인,
`use-widget.ts` 미포함). 직전 라운드(`14_56_27` testing)의 W1(주석 "유일성" 주장 부정확)·W2(신규 테스트 전제

## 요약

이번 diff(전제 단언 2건 + 리드 코멘트 정정, 프로덕션 0줄)는 유지보수성 관점에서 순수 개선이다 — 새 어휘를
만들지 않고 파일이 이미 확립한 "전제 고정" 라벨·`{ result }` 구조분해·`BLOCKED`/`ALLOWED` 표기 컨벤션을
그대로 따랐고, 이전 라운드가 남긴 부정확한 "유일한 가드" 주장을 스코프 한정 + 명시적 삭제 금지 경고로
교체해 향후 "중복 테스트 정리" 류 리팩터링이 이중 방어선의 한쪽을 실수로 지울 위험을 구체적으로 낮췄다.
질문하신 "주석 밀도가 과한가"에 대해서는 이렇게 판단한다 — **이번 diff 자체는 과하지 않다**: 추가된 5줄
내외는 전부 "왜 이 단언/문구가 존재하는가"를 설명하는 최소 정보이고 장식적 서술이 없으며, 함수 길이·중첩
깊이·매직 넘버·순환 복잡도 측면에서도 이 diff 는 기존 테스트에 단일 단언 한 줄씩을 추가했을 뿐 새 이슈를
만들지 않는다. 다만 **파일 전체의 누적 추세**는 4라운드(현재 시점 기준 사실상 그 이상) 연속 회귀라는
실측된 이력에 의해 매 순간 개별적으로는 정당화돼 왔지만, `09_36_01` 라운드가 이미 예견한 "통합 없는 계속된
증가" 그대로 흘러가고 있다 — 지금 당장 위험하지는 않으나(로직이 여전히 검증 중인 hotspot 이라 지금 축약하면
정보 손실 위험이 더 크다) 안정화가 몇 라운드 더 확인되면(이번 라운드 SUMMARY 가 보고한 "Critical 0 7라운드
연속" 신호가 이미 그 근처다) 라운드별 서사를 문서로 이관하고 코드 주석은 현재 유효한 계약만 남기는 통합
패스를 계획할 시점이 임박했다고 본다.
