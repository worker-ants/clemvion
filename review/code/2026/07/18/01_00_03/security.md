# 보안(Security) 리뷰 — 레이어 가드 테스트 문구 회귀 고정 + 이전 리뷰 산출물 커밋

## 리뷰 범위 확인

실제 코드 변경은 `codebase/frontend/src/lib/__tests__/eslint-layering-guard.test.ts` 1개 파일뿐이다.
변경 내용은 ESLint flat config(`eslint.config.mjs`)의 레이어 역전 가드가 내는 에러 메시지의
진입점(정적/동적 import/require)별 상호 배타성을 검증하는 유닛 테스트 보강이며, 런타임 프로덕션
코드·API·데이터 흐름을 변경하지 않는다. 나머지 13개 파일(`spec/conventions/frontend-layering.md`
1줄 문서 추가, `review/code/**`·`review/consistency/**` 신규 산출물 12개)은 프로젝트 저장 규약에
따른 문서/JSON 기록물이며 실행 코드가 아니다.

## 발견사항

없음.

- **인젝션**: 대상 파일은 문자열 리터럴을 ESLint `Linter#verify`/`ESLint#lintText` 인메모리 API에
  fixture 코드로 전달하는 테스트 코드다. 셸 실행·DB 쿼리·HTML 렌더링·경로 접근이 없어 SQL/커맨드/
  XSS/경로 탐색 인젝션 표면이 존재하지 않는다.
- **하드코딩된 시크릿**: diff 전체(테스트 코드, JSON 상태 파일, 리뷰 마크다운)에 API 키·비밀번호·
  토큰·인증서 패턴 없음. `_retry_state.json`/`meta.json`에 기록된 값은 로컬 worktree 절대경로와
  타임스탬프뿐이며 자격증명이 아니다.
- **인증/인가**: 인증·세션·권한 검증 로직과 무관한 테스트/문서 변경.
- **입력 검증**: 사용자로부터 오는 입력이 없다(빌드타임 lint 테스트 fixture는 개발자가 작성한
  상수 문자열).
- **OWASP Top 10**: 해당 사항 없음.
- **암호화**: 해시/암호화/평문 전송과 무관.
- **에러 처리**: 테스트가 검증하는 ESLint 에러 메시지는 계층 라벨과 규약 문서 링크만 담으며
  스택 트레이스·내부 경로·비밀 정보를 노출하지 않는다. `layeringErrors()`의 `throw new Error(...)`
  (fixture 파싱 실패·가드 fail-open 시)도 로컬 테스트 실행 중에만 발생하는 개발 시점 진단
  메시지로, 프로덕션 에러 경로와 무관하다.
- **의존성 보안**: 신규/변경 의존성 없음. 기존 `eslint`/`vitest`만 사용하며 이번 diff로 버전
  변경 없음.

## 요약

이번 변경은 프론트엔드 ESLint 레이어 가드의 에러 메시지 회귀 테스트를 보강하는 순수 테스트/문서
변경으로, 사용자 입력 처리·인증/인가·암호화·네트워크 통신 등 보안 관련 표면을 전혀 건드리지
않는다. 하드코딩된 시크릿, 인젝션 벡터, 취약한 의존성, 민감정보 노출 에러 처리 중 어느 것도
발견되지 않았다.
