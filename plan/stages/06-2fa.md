# Stage 6 · 2FA (TOTP)

## 배경

`NF-SC-10` — Two-Factor Authentication 권장 요구사항. 미구현. TOTP(Google Authenticator 호환) 기반으로 도입한다.

## 설계

### 플로우

1. 사용자가 `/profile`에서 "2단계 인증 설정" 클릭
2. 서버가 `otpauth://` URL + QR 코드 + secret 반환, `user.totp_secret_pending` 저장 (미검증 상태)
3. 사용자가 Authenticator 앱에서 발급된 6자리 코드 입력
4. 서버가 검증 성공 시 `totp_enabled = true`로 전환, 복구 코드 10개 발급 후 일회성 표시
5. 이후 로그인에서 비밀번호 성공 → `/login/verify` 화면에서 TOTP 코드 요구
6. 복구 코드로 2FA 우회 가능 (일회성)

### 데이터 모델

`users` 테이블 확장:
- `totp_secret` VARCHAR NULL (암호화 저장)
- `totp_enabled` BOOLEAN DEFAULT false
- `totp_recovery_codes` TEXT[] NULL (해시 저장)

### API

- `POST /api/v1/auth/2fa/setup` — secret 발급
- `POST /api/v1/auth/2fa/verify` — 검증 + 활성화
- `POST /api/v1/auth/2fa/disable` — 비밀번호 재확인 후 비활성
- `POST /api/v1/auth/login/totp` — 로그인 2단계 검증
