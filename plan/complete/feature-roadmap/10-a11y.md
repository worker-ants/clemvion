# Stage 10 · 접근성 (WCAG 2.1 AA + 스크린 리더)

## 배경

`NF-A11Y-01`, `NF-A11Y-03` 모두 미구현. UI 전반 접근성 감사 + 보강.

## 설계

### 감사 체크리스트

1. **키보드 네비게이션**: Tab 순서, 포커스 트랩(모달·드로어), Esc 닫기
2. **ARIA**: role, aria-label, aria-labelledby, aria-describedby
3. **대비비**: WCAG AA 기준 4.5:1 이상 — Tailwind 색 조합 검증
4. **스크린 리더 흐름**: h1 하나, 헤딩 위계, landmark(nav/main/aside)
5. **폼**: label 연결, 에러 aria-invalid + aria-errormessage
6. **동적 콘텐츠**: aria-live 영역(실행 진행 상태, Toast)
7. **대체 텍스트**: 모든 아이콘에 aria-hidden 또는 의미 전달

### 도구

- `@axe-core/playwright` e2e 자동 감사
- 수동: VoiceOver(macOS)로 주요 화면 확인

### 영향받는 파일

- 거의 모든 `frontend/src/components/**` 점검
- 특히 Radix 기반 컴포넌트는 이미 ARIA 잘 갖춰져 있으므로 커스텀 UI 중심으로 감사
