## 1. 개요

노드는 워크플로우의 기본 구성 단위이다. 각 노드는 특정 기능을 수행하며, 입력 포트로 데이터를 받아 처리한 후 출력 포트로 결과를 전달한다. 노드는 **Logic**, **Flow**, **AI** 세 가지 카테고리로 분류된다.

---

## 2. 노드 공통 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-CM-01 | 모든 노드는 최소 1개의 입력 포트와 1개의 출력 포트를 가짐. 단, 워크플로우의 시작 노드(트리거 직후 첫 노드)는 입력 포트가 트리거로부터 데이터를 수신하는 용도로 사용됨 | 필수 |
| ND-CM-02 | 노드 이름(레이블) 사용자 지정 가능 | 필수 |
| ND-CM-03 | 노드 활성/비활성 토글 — 비활성 시 실행에서 건너뜀 | 필수 |
| ND-CM-04 | 노드 실행 시 입출력 데이터를 기록 (디버깅용) | 필수 |
| ND-CM-05 | 노드 에러 발생 시 에러 처리 정책 선택 (중단/건너뛰기/기본값) | 필수 |
| ND-CM-06 | 노드 설명/메모 필드 | 권장 |
| ND-CM-07 | 노드별 고유 아이콘 및 색상 구분 (카테고리별) | 필수 |

## 3. Logic 노드 (11종)

로직 노드는 데이터 흐름의 분기, 반복, 변수 관리 등 프로그래밍적 제어를 담당한다.

### 3.1 If/Else

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-IF-01 | 조건식을 평가하여 True/False 분기 | 필수 |
| ND-IF-02 | 다양한 비교 연산자 지원 (==, !=, >, <, >=, <=, contains, startsWith, endsWith 등) | 필수 |
| ND-IF-03 | AND/OR 조합으로 복합 조건 구성 | 필수 |
| ND-IF-04 | 출력 포트 2개 (True 경로, False 경로) | 필수 |
| ND-IF-05 | 이전 노드 출력 데이터 필드를 조건식에서 참조 | 필수 |

### 3.2 Switch

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-SW-01 | 입력 값에 따라 N개의 경로 중 하나로 분기 | 필수 |
| ND-SW-02 | 각 케이스에 조건 또는 값 매칭 설정 | 필수 |
| ND-SW-03 | Default(기본) 경로 설정 | 필수 |
| ND-SW-04 | 출력 포트 수를 동적으로 추가/제거 | 필수 |

### 3.3 Loop

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-LP-01 | 지정된 횟수만큼 내부 노드 그룹을 반복 실행 | 필수 |
| ND-LP-02 | 반복 횟수 설정 (고정 숫자 또는 표현식) | 필수 |
| ND-LP-03 | 반복 중 현재 인덱스를 내부 노드에서 참조 가능 | 필수 |
| ND-LP-04 | Break 조건 설정 (조기 종료) | 필수 |
| ND-LP-05 | 최대 반복 횟수 제한 설정 (무한 루프 방지) | 필수 |

### 3.4 Variable Declaration (변수 선언)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-VD-01 | 워크플로우 실행 컨텍스트에 변수를 선언 | 필수 |
| ND-VD-02 | 변수 이름, 타입, 초기값 설정 | 필수 |
| ND-VD-03 | 지원 타입: String, Number, Boolean, Array, Object | 필수 |
| ND-VD-04 | 선언된 변수는 이후 노드에서 참조 가능 | 필수 |

### 3.5 Variable Modification (변수 수정)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-VM-01 | 이전에 선언된 변수의 값을 수정 | 필수 |
| ND-VM-02 | 대상 변수 선택 (드롭다운 또는 표현식) | 필수 |
| ND-VM-03 | 새 값 설정 (정적 값 또는 표현식) | 필수 |
| ND-VM-04 | 배열/객체 변수에 대한 push, pop, set 등의 조작 | 필수 |

### 3.6 Split

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-SP-01 | 배열 데이터를 개별 항목으로 분리하여 각각 다음 노드로 전달 | 필수 |
| ND-SP-02 | 분리 기준(필드 경로) 설정 | 필수 |
| ND-SP-03 | 원본 데이터의 다른 필드를 각 항목에 병합할지 설정 | 권장 |

### 3.7 Map

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-MP-01 | 배열의 각 항목에 변환 로직을 적용하여 새 배열 생성 | 필수 |
| ND-MP-02 | 변환 표현식 설정 (필드 매핑, 값 변환) | 필수 |
| ND-MP-03 | 출력 배열의 각 항목 구조 미리보기 | 권장 |

### 3.8 ForEach

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-FE-01 | 배열의 각 항목에 대해 하위 워크플로우(노드 그룹)를 순차 실행 | 필수 |
| ND-FE-02 | 현재 항목과 인덱스를 하위 노드에서 참조 가능 | 필수 |
| ND-FE-03 | 실행 결과를 배열로 수집 | 필수 |
| ND-FE-04 | 에러 발생 시 처리 정책 (중단/건너뛰기/계속) | 필수 |

### 3.9 Parallel

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-PL-01 | 여러 분기를 동시에(병렬로) 실행 | 필수 |
| ND-PL-02 | 출력 포트 수를 동적으로 추가/제거 | 필수 |
| ND-PL-03 | 모든 분기 완료 후 결과 합산 또는 개별 전달 | 필수 |
| ND-PL-04 | 동시 실행 제한 수 설정 (리소스 관리) | 권장 |

### 3.10 Merge

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-MG-01 | 여러 입력 경로의 데이터를 하나로 합침 | 필수 |
| ND-MG-02 | 입력 포트 수를 동적으로 추가/제거 | 필수 |
| ND-MG-03 | 병합 전략 선택 (대기 후 합치기, 먼저 도착한 것만 등) | 필수 |
| ND-MG-04 | 병합 결과 데이터 구조 설정 (배열, 객체 등) | 필수 |

### 3.11 Background

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| ND-BG-01 | 하위 노드 그룹을 백그라운드로 실행 (메인 흐름을 블로킹하지 않음) | 필수 |
| ND-BG-02 | 메인 흐름은 즉시 다음 노드로 진행 | 필수 |
| ND-BG-03 | 백그라운드 실행 완료/실패 알림 설정 | 필수 |
| ND-BG-04 | 백그라운드 실행 상태 모니터링 | 필수 |
