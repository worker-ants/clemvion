이 페이지에서는 워크플로우가 **언제 저장되는지**, 저장이 **버전**과 어떤 관계인지, 그리고 워크플로우를 **JSON 파일**로 내보내고 다시 가져오는 방법을 정리해요. 스스로 백업을 남기거나, 다른 워크스페이스로 워크플로우를 옮기고 싶을 때 이 페이지를 참고해요.

## 저장하기

- **수동 저장**: 헤더의 `Save` 버튼을 클릭하거나 `Ctrl+S`(Mac은 `Cmd+S`)를 누르면 지금 캔버스 상태가 바로 저장돼요. 저장할 변경 사항이 없으면 버튼이 비활성화돼요.
- **실행 시 자동 저장**: `실행` 버튼(전체 실행이든 선택한 노드부터 실행이든)을 눌렀을 때 저장하지 않은 변경 사항이 남아 있으면, 에디터가 실행 직전에 그 내용을 먼저 자동으로 저장해요. 그래서 실행은 항상 캔버스의 최신 상태를 기준으로 진행돼요.
- 지금 저장 상태는 헤더의 작은 문구로 바로 확인할 수 있어요: **저장 중...** → **저장되지 않은 변경 사항** → **저장됨** 순서로 바뀌어요.

<Callout type="tip">
큰 변경을 마친 뒤에는 헤더 문구가 "저장됨"으로 바뀌는 걸 확인하는 습관을 들이면, 창을 닫거나 [워크플로우 목록](/workflows)으로 돌아가기 전에 작업이 안전하게 저장됐는지 눈으로 바로 알 수 있어요.
</Callout>

<Callout type="note">
아직 해결하지 않은 그래프 오류가 있으면 `Save` 버튼이 잠시 비활성화돼요. 버튼에 마우스를 올리면 어떤 문제인지 안내가 떠요. 오류·경고 배지의 의미는 [유효성 오류](/docs/05-run-and-debug/validation-errors)에서 자세히 다뤄요.
</Callout>

## 저장과 버전의 관계

버전은 아무 때나 쌓이지 않아요. **실제로 저장이 일어나는 순간에만** 그 시점의 스냅샷이 새 버전으로 함께 만들어져요.

- `Save` 버튼이나 `Ctrl+S`로 수동 저장할 때 — 저장할 변경 사항이 있을 때만 새 버전이 생겨요.
- 실행 직전에 자동으로 저장이 일어날 때 — 이때도 그 순간의 상태가 새 버전으로 남아요.

각 버전에는 무엇이 바뀌었는지 자동으로 요약한 변경 요약이 함께 기록돼요. 버전 목록 확인, 두 시점 비교, 예전 상태로 되돌리기는 [버전 히스토리](/docs/05-run-and-debug/version-history)에서 자세히 다뤄요.

## JSON으로 내보내기

워크플로우를 JSON 파일로 내려받아 백업하거나, 동료에게 전달하거나, 다른 워크스페이스로 옮길 때 써요. 내보내기는 두 곳에서 똑같이 할 수 있어요.

<Steps>
  <Step>에디터 안에서: 헤더 우측의 `⋮` (더보기) 메뉴를 열고 **내보내기**를 클릭해요.</Step>
  <Step>[워크플로우 목록](/workflows)에서: 원하는 워크플로우 행의 `⋯` 메뉴를 열고 **내보내기**를 클릭해요.</Step>
  <Step>워크플로우 이름을 파일명으로 하는 `.json` 파일이 바로 다운로드돼요.</Step>
</Steps>

내보낸 JSON에는 워크플로우 이름·설명·태그·설정과 노드·연결선 구성이 담겨요. 실행 이력이나 버전 기록은 담기지 않아요 — 내보내기는 **캔버스 구조**를 옮기기 위한 기능이에요.

## JSON 가져오기

<Steps>
  <Step>[워크플로우 목록](/workflows) 페이지 상단의 **워크플로우 가져오기** 버튼을 클릭해요.</Step>
  <Step>내보내둔 `.json` 파일을 선택해요.</Step>
  <Step>가져오기가 끝나면 목록에 새 워크플로우 항목이 추가돼요.</Step>
</Steps>

가져오기는 기존 워크플로우를 덮어쓰지 않고, **항상 새 워크플로우로** 추가돼요. 파일이 손상됐거나 형식이 올바르지 않으면 에러 안내가 떠요.

<Callout type="note">
**워크플로우 가져오기** 버튼은 에디터 이상 권한을 가진 멤버에게만 보여요.
</Callout>

다른 워크스페이스로 워크플로우를 옮기고 싶다면, [워크스페이스를 전환](/docs/07-workspace-and-team/workspaces-and-members)한 뒤 같은 파일을 가져오면 그 워크스페이스에 새 워크플로우로 추가돼요.
