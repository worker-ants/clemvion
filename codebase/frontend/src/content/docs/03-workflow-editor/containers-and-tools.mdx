이 페이지에서는 여러 노드를 하나로 묶어 반복 실행하는 **컨테이너 노드**와, AI Agent 노드가 도구를 연결하는 방식을 정리해요. 컨테이너는 Loop·ForEach·Map처럼 자식 노드를 안에서 반복 실행하는 노드이고, AI Agent의 도구는 캔버스 위의 별도 영역이 아니라 노드 설정값으로 관리돼요.

## 컨테이너 노드

**Loop**, **ForEach**, **Map** 세 노드가 컨테이너예요. 컨테이너는 자식 노드를 `body`에서 실행하고, 반복마다 나온 결과를 `emit`으로 모아 `done` 포트로 내보내요. (Background 노드는 실행 흐름은 비슷하지만 컨테이너 박스를 쓰지 않고 `background` 포트로 평면적으로 연결돼요 — 이 페이지의 컨테이너 규칙과는 무관해요.)

### 시각적 표현

컨테이너 노드는 다른 노드와 **같은 크기의 카드**로 그려져요. 자식 노드를 감싸는 큰 박스가 아니라, 헤더에 반복 조건 요약(예: `10x · break condition`)이 표시되는 일반 노드 카드예요. 자식 노드는 캔버스 어디에나 자유롭게 놓을 수 있고, 대신 자식 노드 헤더 바로 아래에 `in <컨테이너 이름>` 배지가 붙어서 소속을 알려줘요.

### 자식으로 편입되는 방법

설정 패널에 자식을 지정하는 별도 입력란은 없어요. 소속은 **오직 연결선으로만** 정해지고, 연결이 바뀔 때마다 자동으로 다시 계산돼요.

<Steps>
<Step>컨테이너의 `body` 포트에서 노드로 연결하면, 그 노드가 컨테이너의 자식으로 강제 편입돼요. 이미 다른 컨테이너의 자식이라면 연결이 거부되고, 먼저 그 컨테이너에서 떼어내라는 경고가 떠요.</Step>
<Step>자식 노드에서 또 다른 노드로 계속 연결하면(체인), 다음 노드도 아직 어떤 컨테이너에도 속하지 않은 경우 같은 컨테이너로 자동 편입돼요. 두 노드가 이미 서로 다른 컨테이너에 속해 있다면 조용히 그대로 유지돼요.</Step>
<Step>체인의 마지막 노드에서 컨테이너의 `emit` 포트로 연결하면, 그 노드가 매 반복의 결과를 만드는 자식으로 지정돼요. 역시 이미 다른 컨테이너의 자식이면 연결이 거부돼요.</Step>
</Steps>

`body` 연결을 끊으면 그 노드의 소속이 자동으로 풀려요(단, `emit`에 남아 있거나 체인 중간에 있으면 그 규칙으로 유지돼요). 컨테이너나 자식 노드를 삭제해도 남은 연결선을 기준으로 소속이 다시 계산되므로 어긋난 소속 정보가 남지 않아요. 워크플로우를 다시 열 때도 저장된 연결선을 기준으로 같은 계산이 실행돼 정합성이 자동으로 맞춰져요.

<Callout type="tip">
드래그 앤 드롭으로 노드를 컨테이너 안에 넣는 방식은 없어요. 소속을 바꾸려면 연결선을 다시 이어야 해요.
</Callout>

## 중첩

컨테이너 안에 다른 컨테이너를 배치할 수 있어요. 안쪽 컨테이너도 바깥 컨테이너의 `body`/`emit` 규칙을 그대로 따르는 자식이고, 안쪽 컨테이너 헤더에도 바깥 컨테이너를 가리키는 소속 배지가 함께 붙어요. 별도의 중첩 단계 표시나 배경 색 구분은 없고, 소속 배지를 따라가며 몇 겹인지 확인해요.

<Callout type="note">
Parallel 노드도 여러 분기를 갖지만 컨테이너가 아니라 branch 포트를 여러 개 갖는 일반 노드예요. Parallel을 중첩할 때 지켜야 하는 별도의 깊이 제한은 [검증 오류](/docs/05-run-and-debug/validation-errors)에서 다뤄요.
</Callout>

## 컨테이너 삭제

자식이 있는 컨테이너를 지우려 하면 확인 창이 떠요. 삭제 버튼(✕), 우클릭 메뉴의 "삭제", `Delete`/`Backspace` 키 세 가지 방법 모두 같은 확인 창을 거쳐요.

자식 노드가 없는 빈 컨테이너는 확인 창 없이 바로 삭제돼요. 확인 창에서 기본으로 선택된 옵션은 되돌리기 쉬운 **그룹 해제**예요.

<Callout type="note">
여러 노드를 한꺼번에 선택해 삭제할 때 그중 자식이 있는 컨테이너가 섞여 있으면, 그 컨테이너만 확인 창으로 따로 처리되고 나머지 노드는 그대로 삭제돼요.
</Callout>

일반 노드의 삭제 버튼·우클릭 메뉴·단축키 같은 공통 삭제 방법은 [노드 편집과 정리](/docs/03-workflow-editor/editing-nodes)에서 다뤄요.

## AI Agent와 도구 연결

지금은 캔버스에 AI Agent 전용 도구 영역이 따로 표시되지 않아요. AI Agent가 호출할 도구는 캔버스에서 노드를 끌어와 붙이는 방식이 아니라 **노드 설정 패널**에서 관리해요.

- **지식 저장소(Knowledge Base) 연결** — 설정 패널에서 지식 저장소를 선택하면 RAG 검색 도구로 자동 등록돼요.
- **MCP 서버 연결** — 설정 패널에서 MCP 서버를 연결하면 그 서버가 제공하는 도구가 LLM에게 노출돼요.

두 방식 모두 캔버스에 별도 노드나 연결선으로 나타나지 않고, AI Agent 노드 자체의 설정값으로 존재해요. 그래서 컨테이너처럼 다른 노드를 캔버스에서 끌어와 소속시키는 조작은 필요 없고, AI Agent 노드를 삭제해도 컨테이너 삭제 같은 확인 창은 뜨지 않아요 — 도구가 별도의 캔버스 노드가 아니라 설정값이라 삭제로 함께 잃을 다른 노드가 없기 때문이에요.

<Callout type="tip">
지식 저장소·MCP 서버 필드별 옵션은 [AI 노드](/docs/02-nodes/ai)에서, 지식 저장소 자체를 만들고 관리하는 방법은 [지식 저장소](/docs/06-integrations-and-config/knowledge-base)에서, MCP 서버 등록은 [MCP 서버 통합](/docs/06-integrations-and-config/mcp-servers)에서 다뤄요.
</Callout>

## 컨테이너 안에서의 연결선

컨테이너 경계는 연결선이 직접 넘나들 수 없어요. 컨테이너 바깥의 노드가 안쪽 노드로 바로 연결하거나, 안쪽 노드가 바깥 노드로 바로 연결하는 것 모두 금지예요 — 반드시 컨테이너의 `body`/`emit` 포트를 거쳐야 해요. 컨테이너 내부는 반복 구조라 전체 워크플로우의 순환 검사에서 따로 취급되고, 중첩된 컨테이너는 각 단계마다 독립적으로 검증돼요.

연결선의 색상·유효성 규칙·끊어진 연결 자동 정리 같은 일반 규칙은 [노드 연결하기](/docs/03-workflow-editor/connecting-nodes)에서 확인해요.

## 팁 & 참고

- 컨테이너 노드(Loop/ForEach/Map)의 필드별 설정과 실행 모델은 [로직 노드](/docs/02-nodes/logic)에서 자세히 다뤄요.
- 소속을 확인하고 싶다면 자식 노드 헤더 아래의 `in <컨테이너 이름>` 배지를 확인해요. 배지가 없으면 최상위 노드예요.
- 컨테이너 소속을 바꾸고 싶으면 드래그가 아니라 연결선을 다시 연결해요 — `body`/`emit`에서 떼어내면 소속도 함께 풀려요.
- AI Agent가 호출하는 도구 목록·최대 도구 호출 횟수 같은 세부 필드는 [AI 노드](/docs/02-nodes/ai)에서 확인해요.
- 에디터 전체 레이아웃과 각 영역의 역할은 [워크플로우 에디터 개요](/docs/03-workflow-editor/overview)에서 먼저 살펴보면 이해가 쉬워요.
