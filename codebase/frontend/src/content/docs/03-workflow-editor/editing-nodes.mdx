이 페이지에서는 캔버스에 이미 놓인 노드를 어떻게 선택하고, 옮기고, 복사·복제하고, 지우는지 정리해요. 삭제하지 않고 실행에서만 잠시 건너뛰고 싶을 때 쓰는 **비활성화** 토글과, 노드 카드에 표시되는 상태·설정 요약을 읽는 법도 함께 다뤄요. 캔버스에 노드를 새로 추가하는 방법은 [캔버스 다루기](/docs/03-workflow-editor/canvas-basics)에서, 노드끼리 연결선으로 잇는 방법은 [노드 연결하기](/docs/03-workflow-editor/connecting-nodes)에서 확인해요.

## 노드 선택하기

- **클릭**: 노드를 클릭하면 그 노드 하나만 선택돼요. 이전에 선택했던 노드는 선택이 풀려요.
- **Shift + 클릭**: 이미 선택된 노드 목록에 추가하거나, 다시 누르면 선택에서 빼요.
- **빈 영역 드래그**: 빈 캔버스를 드래그하면 사각형 선택 영역(lasso)이 생기고, 영역에 걸친 노드가 모두 선택돼요.
- **Ctrl + A**: 워크플로우의 모든 노드를 한 번에 선택해요. 빈 캔버스 우클릭 메뉴의 "전체 선택"과 같은 동작이에요.
- **Escape**: 선택을 해제해요. 단, 실행 결과 드로어에 포커스가 있는 상태라면 Escape는 먼저 캔버스로 포커스를 돌려놓는 데 쓰이고, 그 다음에 눌러야 선택이 풀려요. 실행 결과 드로어는 [실행 결과 조회](/docs/05-run-and-debug/run-results)에서 다뤄요.
- **빈 영역 클릭**: 선택이 풀리고, 열려 있던 설정 패널도 닫혀요.

## 이동·복사·복제

노드를 드래그하면 원하는 위치로 옮길 수 있어요. 여러 노드를 선택한 상태에서 그중 하나를 드래그하면 선택된 노드 전부가 함께 움직여요.

**복사·붙여넣기 (Ctrl+C / Ctrl+V)** 는 선택한 노드 묶음을 그대로 재사용하고 싶을 때 써요.

- `Ctrl+C`: 선택한 노드와, 양 끝 노드가 **모두** 선택 안에 포함된 연결선만 함께 복사해요. 선택 밖 노드로 이어지는 연결선은 복사되지 않아요.
- `Ctrl+V`: 복사한 노드를 원본 위치에서 오른쪽·아래로 살짝 떨어진 자리에 붙여넣어요. 이름이 겹치지 않도록 자동으로 고유한 레이블이 붙고, 내부 연결선도 새 노드끼리 다시 연결돼요. 캔버스를 우클릭해 나오는 메뉴의 "붙여넣기"는 클릭한 위치에 붙여 넣는다는 점이 달라요.

**복제 (Ctrl+D)** 는 클립보드를 건드리지 않고 선택한 노드를 그 자리에서 바로 하나 더 만들어요. 노드를 우클릭해 나오는 메뉴의 "복제"도 같은 동작이에요.

<Callout type="note">
워크플로우를 시작하는 Manual Trigger 노드는 워크플로우당 한 개만 존재할 수 있어서 복제할 수 없어요. 우클릭 메뉴에서 "복제" 항목이 비활성 상태로 보여요.
</Callout>

## 노드 삭제

노드를 지우는 방법은 두 가지예요.

- **Delete 또는 Backspace 키**: 선택한 노드를 지워요.
- **노드 우상단의 ✕ 버튼**: 노드에 마우스를 올리거나 노드를 선택하면 나타나요. 클릭하면 그 노드만 바로 지워요.

두 방법 모두 노드에 연결된 연결선도 함께 사라져요. 실수로 지웠다면 `Ctrl+Z`로 되돌릴 수 있어요.

### 자식 노드가 있는 컨테이너를 지울 때

Loop·ForEach·Map처럼 안에 다른 노드를 담을 수 있는 컨테이너 노드를 지우려고 하면, 자식 노드가 있는지에 따라 동작이 갈려요.

<Steps>
<Step>자식 노드가 없는 컨테이너와 일반 노드는 확인 없이 바로 지워져요.</Step>
<Step>자식 노드가 있는 컨테이너를 지우려 하면 확인 창이 떠서 두 가지 중 하나를 고르게 해요.</Step>
<Step>**그룹 해제**(기본 선택): 컨테이너만 없어지고, 자식 노드는 최상위 노드로 남아요. 자식 노드끼리의 연결선은 그대로 유지돼요.</Step>
<Step>**컨테이너와 모든 자식 삭제**: 컨테이너와 그 자식 노드 전부, 관련 연결선까지 한꺼번에 지워져요.</Step>
</Steps>

이 확인 창은 ✕ 버튼·우클릭 메뉴의 "삭제"·Delete 키 중 어떤 방법으로 지우든 똑같이 나타나요. 컨테이너의 구조와 멤버십 규칙은 [컨테이너와 AI Agent 도구](/docs/03-workflow-editor/containers-and-tools)에서 더 자세히 다뤄요.

<Callout type="note">
Manual Trigger 노드는 워크플로우의 시작점이라 삭제할 수 없어요. 이 노드에는 ✕ 삭제 버튼 자체가 나타나지 않아요.
</Callout>

## 노드 비활성화

노드를 지우지 않고 **실행에서만 건너뛰고** 싶을 때는 비활성화를 써요. 노드와 연결선은 캔버스에 그대로 남지만, 워크플로우를 실행하면 이 노드는 건너뛰고 다음으로 넘어가요.

켜고 끄는 방법은 두 가지예요.

- 노드를 선택해서 [설정 패널](/docs/03-workflow-editor/settings-panel)을 열고, 하단의 "이 노드 비활성화" 체크박스를 켜요.
- 노드를 우클릭해서 나오는 메뉴에서 "비활성화"(다시 켜려면 "활성화")를 선택해요.

비활성화된 노드는 캔버스에서 **흐리게** 표시돼서 한눈에 구분할 수 있어요.

<Callout type="tip">
삭제와 비활성화는 다른 목적이에요. 노드와 연결을 그대로 두고 잠깐만 워크플로우 흐름에서 빼고 싶다면 비활성화를, 완전히 없애고 싶다면 삭제를 써요.
</Callout>

## 노드 이름과 메모

[설정 패널](/docs/03-workflow-editor/settings-panel)의 Name 필드로 노드에 표시할 레이블을 바꿀 수 있어요. 여기서 정한 이름이 캔버스의 노드 카드 둘째 줄에 그대로 보여요.

- 노드 이름은 워크플로우 안에서 서로 겹칠 수 없어요. 이미 쓰인 이름으로 바꾸려 하면 저장이 막히고 빨간 안내 문구가 떠요.
- Notes 필드에는 이 노드에 대한 메모를 자유롭게 남길 수 있어요. 다른 협업자가 왜 이렇게 설정했는지 이해하는 데 도움이 돼요.

필드별 세부 설정과 표현식 입력 방법은 [노드 설정 패널](/docs/03-workflow-editor/settings-panel)에서 이어서 다뤄요.

## 노드 상태와 설정 요약 읽기

노드 카드만 봐도 설정을 미리 파악하고, 실행 중에는 진행 상황을 알 수 있어요.

### 설정 요약

노드 본체 세 번째 줄에 핵심 설정을 요약해서 보여줘요. 예를 들어 HTTP Request 노드라면 `GET https://api.example.c...` 처럼 메서드와 URL이 보여요. 텍스트가 길면 말줄임표로 잘리고, 마우스를 올리면 전체 텍스트가 툴팁으로 떠요. 설정을 바꾸면 이 요약도 실시간으로 갱신돼요. 다만 캔버스를 50% 미만으로 축소하면 요약 줄은 숨겨지고 아이콘·유형명·이름만 남아요.

### 미설정 경고

필수 설정이 하나라도 비어 있으면 노드 헤더 오른쪽에 노란 삼각형 경고 아이콘이 떠요. 마우스를 올리면 어떤 항목이 비어 있는지 구체적으로 안내해요. 예를 들어 HTTP Request 노드는 URL을 입력하지 않으면 경고가 뜨고, Send Email 노드는 받는 사람을 지정하지 않으면 경고가 떠요. 안내 문구는 노드 종류마다 달라요. 필수 항목이 아예 없는 Manual Trigger 노드는 이 경고가 뜨지 않아요.

### 실행 상태 표시

워크플로우를 실행하면 각 노드에 지금 어떤 단계인지 표시돼요.

| 상태 | 화면 표시 |
| --- | --- |
| 대기 중 | 특별한 표시 없음 |
| 실행 중 | 파란 테두리가 펄스로 깜빡여요 |
| 성공 | 초록 테두리 + 체크 아이콘 |
| 실행 실패 | 빨간 테두리 + 에러 아이콘. 클릭하면 에러 상세를 볼 수 있어요 |
| 건너뜀 | 회색으로 흐려져요 (비활성화 노드나 조건에 안 맞아 지나간 경로) |

이 표시는 다음 실행을 시작하기 전까지 캔버스에 남아 있어요. 각 노드가 실제로 무엇을 입력받고 출력했는지, 어디서 에러가 났는지처럼 더 자세한 내용은 [실행 결과 조회](/docs/05-run-and-debug/run-results)에서 확인해요. 워크플로우를 실행하는 방법 자체는 [워크플로우 실행하기](/docs/05-run-and-debug/running-a-workflow)를 참고해요.

## 팁 & 참고

- 노드를 여러 개 함께 옮기거나 지우고 싶다면 먼저 Shift+클릭이나 드래그로 선택 범위를 넓힌 뒤 작업하면 편해요.
- 붙여넣기·복제로 늘어난 노드는 이름이 자동으로 구분되지만, 나중에 헷갈리지 않도록 [설정 패널](/docs/03-workflow-editor/settings-panel)에서 바로 알아보기 쉬운 이름으로 바꿔 두는 걸 추천해요.
- 큰 워크플로우를 정리할 때는 컨테이너로 묶는 방법이 유용해요. 자세한 멤버십 규칙은 [컨테이너와 AI Agent 도구](/docs/03-workflow-editor/containers-and-tools)에서 다뤄요.
- 노드 추가·연결까지 포함한 캔버스 전체 흐름은 [워크플로우 에디터 개요](/docs/03-workflow-editor/overview)에서 큰 그림으로 볼 수 있어요.
