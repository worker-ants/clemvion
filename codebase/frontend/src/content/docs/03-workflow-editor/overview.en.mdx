This page walks through what the workflow editor is and how its screen is laid out. The workflow editor is where you place nodes on a canvas and link them with connections to build automation logic visually. The detailed mechanics of each region are covered in the pages that follow.

## The editor's five regions

- **Header** (top) — Click the back arrow (←) to return to the [workflow list](/workflows). Click the workflow name in the center to rename it inline. The header also shows the save-status text, Undo/Redo, the `🤖` AI Assistant toggle, the `Save` button, the `▶ Run` button (with a dropdown for run options), and the `⋮` more menu (version history, execution history, export, delete).
- **Node palette** (left) — Lists nodes by category, with search and a recently-used section. Covered in detail in [Working the canvas](/docs/03-workflow-editor/canvas-basics).
- **Canvas** (center) — The workspace where you place nodes and link them with connections. Node manipulation is covered in [Editing and organizing nodes](/docs/03-workflow-editor/editing-nodes), and connections in [Connecting nodes](/docs/03-workflow-editor/connecting-nodes).
- **Settings panel / AI Assistant panel** (right, conditional) — Clicking a node opens the settings panel; clicking `🤖` opens the AI Assistant panel. They share the same slot, so only one is open at a time. See [Node settings panel](/docs/03-workflow-editor/settings-panel) and [AI Assistant overview](/docs/03-workflow-editor/ai-assistant) for details.
- **Run Results drawer** (bottom, appears while running) — Opens along the bottom when you run a workflow, showing per-node execution results. See [Running a workflow](/docs/05-run-and-debug/running-a-workflow) for how to trigger a run.

<Callout type="note">The settings panel and the AI Assistant panel share the same right-hand slot. Selecting a node closes the assistant, and opening the assistant clears the node selection.</Callout>

## Entering and leaving the editor

- Clicking an item in the [workflow list](/workflows) opens that workflow's editor.
- Clicking **Create Workflow** in the list takes you straight into an empty canvas. A single Manual Trigger node is already placed there, so you can immediately attach the next node.
- Clicking the back arrow (←) takes you back to the [workflow list](/workflows).

When you have unsaved changes, the header's status text says "Unsaved changes" ("Saving..." while a save is in flight, "Saved" once it finishes). Check this indicator and save before you leave for the list.

## Saving, at a glance

- **Manual save**: Use the header's `Save` button or `Ctrl+S` (`Cmd+S` on Mac) to save immediately. The button is disabled when there's nothing to save.
- **Auto-save before a run**: When you run the workflow with `▶ Run`, any unsaved changes are saved first, right before execution — so a run always uses the latest canvas.
- The relationship between saving and version history, plus importing/exporting workflow JSON, is covered in [Saving, auto-save, and import/export](/docs/03-workflow-editor/saving-and-sharing).

## Starting from an empty canvas

A new workflow starts with just a trigger node. If there's no node attached after the trigger yet, a "Let's finish this workflow" prompt card appears in the top-right corner of the canvas.

- The card shows a 3-step checklist (① drag the next node from the palette ② connect it to the trigger's output port ③ run it to see the result), a "Learn more" link next to each step, and a "Open starter guide" button that opens [Your first workflow](/docs/01-getting-started/first-workflow).
- The card fades out once you add a first non-trigger node. You can also dismiss it directly with the close button in its top-right corner.
