Telegram 봇을 워크플로우의 대화 인터페이스로 연결하면, 봇에게 메시지를 보내는 것만으로 워크플로우를 시작하거나 실행 중인 워크플로우에 입력을 제공할 수 있어요.

## 1. Telegram 봇 만들기

1. Telegram 앱에서 [@BotFather](https://t.me/botfather)를 검색해 채팅을 시작해요.
2. `/newbot` 명령을 입력하고 안내에 따라 봇 이름과 username을 설정해요.
3. BotFather가 **API Token** (`123456789:ABCdef...` 형태)을 알려줘요. 이 토큰을 안전하게 보관해요.

<Callout type="warn">
Bot API Token은 봇 계정을 완전히 제어할 수 있는 자격증명이에요. 코드나 공개 저장소에 직접 넣지 마세요.
</Callout>

## 2. 워크플로우 Webhook 트리거에 Chat Channel 설정

**GUI 등록 흐름 (권장)**:

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/triggers/page.tsx"
  symbol="chatChannelProvider"
  describes="트리거 생성 dialog 의 Chat Channel 섹션 (Spec Chat Channel §4.1 / 2-trigger-list §2.3.1 R-8)"
/>

1. 좌측 메뉴 → **Triggers** → 우측 상단 **"+ Webhook 트리거 추가"** 클릭
2. 트리거 이름·연결할 워크플로우·인증 방식 입력
3. 폼 하단의 **"Chat Channel 연결 (Telegram 봇)"** 체크박스를 켜기
4. 등장하는 **"신규 Bot Token"** 필드에 BotFather 가 발급한 token 입력 (`123456789:ABC...`)
5. **만들기** 클릭 → 트리거 생성과 동시에 `setWebhook` 자동 호출 → Telegram 봇과 워크플로우 즉시 연결

트리거 상세 drawer 의 **"Chat Channel"** 카드에서 health / UI 매핑 / 안내 메시지 i18n 을 확인·편집할 수 있고, **"Bot Token 재발급"** 버튼으로 토큰 회전도 GUI 에서 처리합니다.

**API 등록 흐름 (자동화 / CI 용)**:

GUI 없이 API 로 등록하려면 트리거 생성 본문에 `chatChannel` 을 포함하세요:

```bash
curl -X POST https://<your-host>/api/triggers \
  -H 'Authorization: Bearer <your-token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "type": "webhook",
    "name": "telegram-bot",
    "workflowId": "<workflow-id>",
    "config": {
      "chatChannel": {
        "provider": "telegram",
        "botToken": "<BotFather에서 받은 token>"
      }
    }
  }'
```

트리거가 활성화되면 자동으로 Telegram `setWebhook` API 가 호출돼 봇이 워크플로우 엔드포인트로 연결돼요. 수동으로 BotFather 에서 webhook 을 설정할 필요가 없어요.

- `botToken` 은 응답에 절대 노출되지 않아요 — secret store 에 AES-256-GCM 으로 암호화 보관되고, 응답에는 `hasBotToken: true` 만 포함돼요.
- `botToken` 형식: `^\d{6,}:[A-Za-z0-9_-]{30,}$` (BotFather 발급 표준). 위반 시 400 `BOT_TOKEN_INVALID`.

## 3. 지원하는 인터랙션 노드

Chat Channel을 통해 아래 인터랙션 노드 유형을 사용할 수 있어요:

| 노드 유형 | Telegram 표현 방식 |
|---|---|
| **AI Multi Turn** | 봇과 자유 텍스트 대화 |
| **Button Presentation** | 인라인 키보드 버튼으로 선택 |
| **Form** | 필드별 순차 텍스트 입력 (연락처 공유, 파일 업로드 포함) |
| **Chart** | monospace 미니 막대 그래프 텍스트 (라벨·값 정렬, 첫 시리즈 표시) |
| **Table** | monospace MarkdownV2 표 (열 너비 자동 정렬, 최대 20행 — 초과 시 "외 N행" 안내) |
| **Carousel** | 카드별 sequential 메시지 (제목·설명·이미지 URL, 최대 10장) |

<Callout type="note">
시각형 노드(Chart, Table, Carousel)는 현재 **MarkdownV2 텍스트 표현**으로 동작해요. 향후 별도 PR 에서 SSR PNG 이미지 발송으로 격상될 예정이에요 (별 plan `chat-channel-visual-ssr-png` 추적).

`uiMapping.visualNode` 옵션 (default `"auto"`):

| 값 | 동작 |
|---|---|
| `"auto"` (default) | 노드별 자동 선택 — chart/table 은 텍스트, carousel 은 카드별 `imageUrl` 있으면 `sendPhoto` else 텍스트 |
| `"text"` | 모든 시각형을 텍스트만 발송 (carousel `imageUrl` 도 무시) |
| `"photo"` | v2 SSR 인프라 도입 후 PNG 이미지 발송. v1 단계에서는 `text` 와 동일하게 동작 (fallback) — 정상 동작이며 health 변경 없음 |

> 구 enum 값 `"text_only"` 은 폐기됐어요. DB 에 남아있는 값은 어댑터가 자동으로 `"text"` 로 변환하므로 기존 트리거는 그대로 동작해요.
</Callout>

## 5. 상태 모니터링

트리거 상세 화면의 `chatChannelHealth` 필드로 연결 상태를 확인해요:

| 값 | 의미 |
|---|---|
| `unknown` | 아직 `setupChannel` 이 호출되지 않았거나 확인 전 |
| `healthy` | 봇 연결 정상 |
| `degraded` | `setupChannel` / `rotate-bot-token` 실패 또는 sendMessage 연속 실패. **자동 비활성화는 하지 않아요** (CCH-SE-01) — 사용자가 명시적으로 활성/비활성 토글해야 해요 |

`chatChannelLastError` 필드에서 마지막 오류 내용을 확인할 수 있어요.

## 6. Bot Token 회전 (single-path)

토큰 변경은 **항상 아래 rotate API 만** 사용해요. PATCH 본문으로 `config.chatChannel.botTokenRef` 를 직접 변경하면 400 `VALIDATION_ERROR` 가 돌아와요 (`details.field='botTokenRef'`).

```bash
curl -X POST https://<your-host>/api/triggers/<trigger-id>/chat-channel/rotate-bot-token \
  -H 'Authorization: Bearer <your-token>' \
  -H 'Content-Type: application/json' \
  -d '{ "newBotToken": "<신규 BotFather token>" }'
```

- 기존 token 은 **24시간 동안 grace 기간** 으로 같이 받아요 (그 사이 진행 중인 update 는 유실 없이 처리). 24시간 후 자동 정리.
- 응답: `{ data: { triggerId, rotatedAt, chatChannelHealth, botIdentity } }`
- 에러: 400 `BOT_TOKEN_INVALID`, 404 `TRIGGER_NOT_FOUND`, 502 `CHAT_CHANNEL_SETUP_FAILED` 등 ([Spec Chat Channel §5.4](https://github.com/worker-ants/clemvion/blob/main/spec/5-system/15-chat-channel.md#54-bot-token-rotation-api-응답-계약))

## 7. 제한 사항

- **1:1(private) 대화만 지원** — 그룹 채팅에서 봇에게 메시지를 보내도 무시돼요. 봇은 안내 메시지(`languageHints.groupChatRefusal`) 만 발송해요.
- **봇이 보낸 메시지는 무시** — 봇 간 루프 방지를 위해 `from.is_bot=true` 인 메시지는 처리하지 않아요.
- **1분당 60건 제한** — 같은 conversation 에서 60건을 초과하면 초과 요청이 무시돼요 (CCH-NF-03).
- **Bot Token 보관** — Bot Token 과 webhook `secret_token` 은 secret store 에 AES-256-GCM 으로 암호화 저장돼요 (CCH-SE-03). DB 에는 ciphertext 만 보관되고, plaintext 는 master key 를 보유한 backend 프로세스 안에서만 일시적으로 복호화돼요. API 응답에는 평문 token 이 절대 노출되지 않고, `hasBotToken: boolean` derived 필드만 포함돼요.

## Rationale

### Webhook 트리거의 갈래로 구현한 이유

Telegram Chat Channel을 별도 트리거 유형으로 만들지 않고 Webhook 트리거의 `config.chatChannel` 갈래로 배치했어요. 이렇게 하면 기존 Webhook 진입점을 재사용해 레이어 추가 없이 최소 표면으로 도입할 수 있고, 향후 Slack·카카오 등 다른 provider 추가 시에도 같은 패턴을 따를 수 있어요.
