## Embedding 탭 — 임베딩 모델 설정

**Embedding** 탭은 지식 저장소가 문서를 벡터로 변환할 때 사용하는 임베딩 모델을 관리해요. 임베딩 설정은 지식 저장소 생성·수정 화면의 임베딩 모델 드롭다운에서 선택해요.

### 지원 프로바이더

임베딩 설정은 Chat 설정과 동일한 프로바이더 구조(OpenAI · Google AI · Azure · Local)를 사용해요. Anthropic 은 임베딩을 지원하지 않아요.

### 임베딩 모델 설정 추가하기

**GUI 흐름**: 좌측 메뉴에서 [모델 설정](/models?tab=embedding)을 클릭한 뒤 **Embedding** 탭을 선택해요. 우측 상단의 `[+ 모델 추가]` 를 눌러요.

## Rerank 탭 — 리랭커 설정

**Rerank** 탭은 지식 저장소의 검색 결과를 2차로 정밀화하는 리랭커(Reranker)를 관리해요. 리랭커 설정을 마친 뒤에는 [지식 저장소](/docs/06-integrations-and-config/knowledge-base)의 Reranking 탭에서 컬렉션별로 활성화할 수 있어요.

### 리랭커란?

벡터 유사도 검색은 빠르지만, 의미적으로 비슷한 청크가 뒤섞여 반환될 수 있어요. **리랭커(cross-encoder 모델)** 는 검색 후처리 단계에서 `(질의, 청크)` 쌍을 직접 비교해 관련도 점수를 재계산하고, 더 정확한 순서로 상위 결과를 골라내요.

주요 특징은 다음과 같아요.

- 검색 시점에만 적용되므로 기존 임베딩을 다시 만들 필요가 없어요.
- 리랭커 오류가 발생해도 벡터 유사도 순서로 안전하게 대체돼요.
- 지식 저장소 단위로 켜거나 끌 수 있어요.

### 기본 리랭커 설정

목록에서 `⭐ 기본 설정` 버튼을 누르면 워크스페이스 기본 리랭커가 지정돼요. 지식 저장소의 Reranking 탭에서 리랭커를 따로 고르지 않으면 이 기본 리랭커가 사용돼요. 기본 리랭커가 없을 때는 해당 지식 저장소의 리랭킹이 비활성 상태로 안전하게 처리돼요.

<Callout type="note">기본 리랭커 변경은 이후 검색부터 반영돼요. 이미 설정된 지식 저장소에는 영향을 주지 않아요.</Callout>

### 리랭커가 쓰이는 곳

등록한 리랭커는 [지식 저장소](/docs/06-integrations-and-config/knowledge-base) 의 Reranking 탭에서 컬렉션별로 활성화해요. 활성화하면 AI Agent 가 해당 지식 저장소를 검색할 때 자동으로 2차 정밀화가 적용돼요.

## 팁 & 참고

- Chat 탭의 기본 프로바이더는 목록에서 `⭐ 기본 설정` 버튼으로 바꿀 수 있어요. 이미 만든 AI 노드에는 영향을 주지 않고, 신규 노드에만 반영돼요.
- Anthropic은 임베딩을 지원하지 않아요. Chat 기본 프로바이더로 Anthropic 을 사용하더라도 지식 저장소 임베딩을 위한 별도 임베딩 설정을 Embedding 탭에 등록해야 해요.
- TEI 서버를 GPU 환경에서 운영 중이라면 `dragonkue/bge-reranker-v2-m3-ko` 모델을 사용하면 한국어 검색 품질이 크게 향상돼요.
- Cohere 리랭커는 인터넷 연결이 필요해요. 클라우드 환경에서 빠르게 시작하기에 좋아요.
- 리랭커를 삭제하면 해당 리랭커를 사용 중인 지식 저장소의 리랭킹이 비활성 상태로 대체돼요.
- AI 노드에서 파라미터를 개별 조정하는 법은 [AI 노드](/docs/02-nodes/ai)에서 설명해요.
- Azure OpenAI는 모델 ID에 배포 이름(deployment name)을 입력해야 해요. 표준 모델 ID를 넣으면 404가 떨어져요.
