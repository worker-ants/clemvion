Slack 봇을 워크플로우의 대화 인터페이스로 연결하면, 봇에게 DM 으로 메시지를 보내거나 slash command 를 입력해서 워크플로우를 시작·조작할 수 있어요.

## 1. Slack 앱 만들기

1. [Slack API 콘솔](https://api.slack.com/apps)에서 **Create New App** → **From scratch** 를 선택해요.
2. 앱 이름과 install 할 workspace 를 선택해요.
3. 생성된 앱의 **Basic Information** 페이지에서 두 가지 자격증명을 확인해요:
   - **Signing Secret** — Slack 이 보내는 요청의 출처 검증에 사용 (`X-Slack-Signature` HMAC key)
   - **Bot User OAuth Token** — `xoxb-...` 로 시작. 봇으로 메시지 발송에 사용 (다음 단계에서 install 후 확인)

<Callout type="warn">
Signing Secret 과 Bot Token 은 봇 계정을 완전히 제어할 수 있는 자격증명이에요. 코드나 공개 저장소에 직접 넣지 마세요. clemvion 은 두 값을 모두 backend AES-256-GCM 으로 암호화해 저장하고, UI 에는 노출하지 않아요.
</Callout>

## 2. 앱 manifest 설정

Slack 앱의 **App Manifest** 페이지에서 다음 항목을 등록해요:

### 2.1 OAuth & Permissions — Bot Token Scopes

- `chat:write` — 봇이 메시지 발송
- `im:history` — DM 메시지 수신
- `im:read` — DM 채널 정보 조회
- `commands` — slash command 처리
- `files:read` — file_shared event 의 첨부 파일 정보 조회 (Form `file` 필드)

### 2.2 Event Subscriptions

- **Enable Events** = On
- **Request URL** = `https://<your-clemvion-domain>/api/hooks/<endpointPath>`
  - `<endpointPath>` 는 다음 단계에서 트리거 생성 시 발급받은 값
  - Slack 이 1회 URL Verification handshake (challenge 요청) 를 보내고 우리 backend 가 자동 응답
- **Subscribe to bot events**:
  - `message.im` — DM 메시지
  - `file_shared` — DM 파일 업로드 (Form `file` 필드용)

### 2.3 Interactivity & Shortcuts

- **Interactivity** = On
- **Request URL** = Event Subscriptions 와 **동일한 URL** (Block Kit button click / modal submit 도 같은 endpoint 로 수신)

### 2.4 Slash Commands (선택 — power user 용)

- Command = `/workflow` (또는 원하는 prefix)
- Request URL = 위와 동일
- Short description = "Workflow assistant"

## 3. clemvion 트리거에 Slack 연결

1. clemvion 의 워크플로우 화면 → **Webhook 트리거** 생성
2. 트리거 설정 패널의 **Chat Channel 연결** 섹션을 활성화
3. **Provider** = `Slack` 선택
4. **Bot Token** 입력 — Slack 앱의 **OAuth & Permissions** 페이지에서 **Install to Workspace** 후 발급된 `xoxb-...` 토큰
5. **Signing Secret** 입력 — Slack 앱 **Basic Information** 페이지의 값
6. 저장 → backend 가 `auth.test` 호출로 봇 식별자 (`bot_id`, `username`, `team_id`) 자동 캐시
7. 발급된 `endpointPath` 를 Slack 앱 manifest 의 Request URL 들에 입력 (2.2 / 2.3)

## 4. 사용 방법

### 4.1 DM 으로 대화

- Slack 에서 봇 이름을 클릭해 DM 시작
- 첫 메시지 송신 → 워크플로우 새 execution 시작 (Telegram `/start` 와 동등)
- AI Multi Turn 노드의 응답은 자동으로 DM 으로 도착
- 사용자 reply 는 그대로 다음 입력으로 전달

### 4.2 Slash command (선택)

- `/workflow` (또는 prefix) — DM 첫 메시지와 동일하게 새 execution 시작
- `/workflow cancel` — 진행 중인 execution 취소
- `/workflow reply <message>` — AI Multi Turn 진행 중 자유 텍스트 reply (DM 메시지 대체)

### 4.3 Button / Form

- Button Presentation 노드의 버튼은 Slack Block Kit `actions block` 으로 자동 변환
- Form 노드는 v1 에서 다단계 텍스트 시퀀스로 진행 (필드별 한 줄 질문 + DM reply 로 답변)
- v2 (후속) 에서 Slack modal (`views.open`) native 지원 예정

## 5. v1 한계 / 알려진 사항

- **DM only** — group chat / channel / mpim 은 미지원 (자동 거부 + 안내 메시지). multi-user 채널 대응은 v2 (multi-user thread) 와 한 묶음.
- **Typing indicator 없음** — Slack Web API 가 server-initiated typing indicator 를 제공 안 함. LLM 응답 latency 가 긴 경우 사용자가 "처리 중" 표시를 못 받음 (v2 후보 plan 에서 placeholder 메시지 + edit 으로 검토).
- **시각형 노드** (Chart / Table / Carousel) — v1 은 mrkdwn monospace 텍스트 fallback. v2 에서 SSR PNG (Telegram 과 공유 plan) 격상.
- **Socket Mode 미지원** — v1 은 Webhook-mode 만. private network 안의 Slack 운영은 v2 plan 에서 검토.

## 6. 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| Slack 이 URL Verification 실패 → "Your URL didn't respond with the value of the challenge parameter" | Signing Secret 미일치 또는 endpoint 가 publicly reachable 아님. clemvion 의 채널 health 가 `degraded` 면 setup 오류 |
| Slack 이 retry 폭주 ("dispatch_failed") | 3초 이내 ack 실패. backend latency 점검 — `chat_channel_health` 가 `degraded` 면 외부 API 호출 자체 실패 가능 |
| Bot 이 DM 에 응답 안 함 | `message.im` event subscription 누락 또는 봇이 conversation 멤버 아님 (DM 은 자동) |
| `invalid_auth` 에러 | Bot Token 만료 — clemvion 트리거 패널의 **Bot Token 재발급** 으로 24h grace 회전 |
