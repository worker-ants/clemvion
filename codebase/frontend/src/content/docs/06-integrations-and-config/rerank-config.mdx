이 페이지에서는 지식 저장소의 검색 결과를 2차로 정밀화하는 **리랭커(Reranker)** 를 등록하고 관리하는 방법을 설명해요. 리랭커는 [모델 설정](/models?tab=rerank)의 **Rerank** 탭에서 관리해요. 리랭커 설정을 마친 뒤에는 [지식 저장소](/docs/06-integrations-and-config/knowledge-base)의 Reranking 탭에서 컬렉션별로 활성화할 수 있어요.

## 리랭커란?

벡터 유사도 검색은 빠르지만, 의미적으로 비슷한 청크가 뒤섞여 반환될 수 있어요. **리랭커(cross-encoder 모델)** 는 검색 후처리 단계에서 `(질의, 청크)` 쌍을 직접 비교해 관련도 점수를 재계산하고, 더 정확한 순서로 상위 결과를 골라내요.

주요 특징은 다음과 같아요.

- 검색 시점에만 적용되므로 기존 임베딩을 다시 만들 필요가 없어요.
- 리랭커 오류가 발생해도 벡터 유사도 순서로 안전하게 대체돼요.
- 지식 저장소 단위로 켜거나 끌 수 있어요.

## 리랭커 추가하기

**GUI 흐름**: 좌측 메뉴에서 [모델 설정](/models?tab=rerank)을 클릭한 뒤 **Rerank** 탭을 선택해요. 우측 상단의 `[+ Add Provider]` 를 눌러요.

  - **Cohere**: `API Key` 필수. Base URL 은 입력하지 않아요.
</Step>
<Step>기본 모델 ID를 직접 입력해요. 예: TEI 의 경우 `dragonkue/bge-reranker-v2-m3-ko` 또는 `bge-reranker-v2-m3`, Cohere 의 경우 `rerank-3.5`.</Step>
<Step>저장해요. 가장 자주 쓰는 설정은 ⭐ **기본 리랭커 설정** 으로 표시해 두면, 지식 저장소에서 리랭커를 별도로 지정하지 않아도 자동으로 선택돼요.</Step>
</Steps>

<Callout type="warn">API Key 는 저장 후 마스킹 처리되어 응답에 돌아와요. 원본 키는 프로바이더 콘솔에서 따로 관리해요.</Callout>

## 기본 리랭커 설정

목록에서 ⭐ **기본 리랭커 설정** 버튼을 누르면 워크스페이스 기본 리랭커가 지정돼요. 지식 저장소의 Reranking 탭에서 Reranker 를 따로 고르지 않으면 이 기본 리랭커가 사용돼요. 기본 리랭커가 없을 때는 해당 지식 저장소의 리랭킹이 비활성 상태로 안전하게 처리돼요.

<Callout type="note">기본 리랭커 변경은 이후 검색부터 반영돼요. 이미 설정된 지식 저장소에는 영향을 주지 않아요.</Callout>

## 리랭커가 쓰이는 곳

등록한 리랭커는 [지식 저장소](/docs/06-integrations-and-config/knowledge-base) 의 Reranking 탭에서 컬렉션별로 활성화해요. 활성화하면 AI Agent 가 해당 지식 저장소를 검색할 때 자동으로 2차 정밀화가 적용돼요.

## 팁 & 참고

- TEI 서버를 운영 중이라면 GPU 가 있는 환경에서 `dragonkue/bge-reranker-v2-m3-ko` 모델을 사용하면 한국어 검색 품질이 크게 향상돼요.
- Cohere 는 인터넷 연결이 필요해요. 클라우드 환경에서 빠르게 시작하기에 좋아요.
- 리랭커를 삭제하면 해당 리랭커를 사용 중인 지식 저장소의 리랭킹이 비활성 상태로 대체돼요.
- 지식 저장소 Reranking 탭의 상세 옵션(후보 풀 크기, 점수 컷)은 [지식 저장소](/docs/06-integrations-and-config/knowledge-base) 에서 설명해요.
