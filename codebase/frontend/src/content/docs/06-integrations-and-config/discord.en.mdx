Connect a Discord bot as your workflow's interface, and users can drive workflows via `/workflow start` slash commands, button clicks, and modal text inputs.

<Callout type="warn">
{/* Internal SoT: spec/5-system/15-chat-channel.md R-CC-13 + spec/4-nodes/7-trigger/providers/discord.md R-D-3 */}
Discord v1 limitation: free-text DM messages are not supported. v1 uses Interactions Webhook only; receiving plain DM text requires a Gateway WebSocket connection, which is out of scope for v1. User reply paths: (a) `/workflow reply <message>` slash command text option, (b) the **Reply** button on AI messages opens a modal TEXT_INPUT (default UX).
</Callout>

## 1. Create a Discord app

1. Go to the [Discord Developer Portal](https://discord.com/developers/applications) → **New Application**
2. From **General Information**, note:
   - **Application ID**
   - **Public Key** — 64-char hex used for ed25519 signature verification on Interactions Webhook
3. **Bot** tab → **Reset Token** to obtain the **Bot Token** (`xxx.yyy.zzz`)

<Callout type="warn">
The Bot Token grants full control over the bot account. Public Key is public info, but clemvion's secret store manages both with backend AES-256-GCM for consistency.
</Callout>

## 2. Interactions Endpoint URL

In the **General Information** page:

- **Interactions Endpoint URL** = `https://<your-clemvion-domain>/api/hooks/<endpointPath>`
- On save, Discord sends a PING (type=1). clemvion responds automatically with `{ type: 1 }` 200
- "Validation Failed" → public key mismatch or endpoint unreachable

## 3. Bot permissions + invite

1. **OAuth2 → URL Generator**
   - **scopes**: `bot`, `applications.commands`
   - **bot permissions**: `Send Messages`, `Read Message History`, `Use Slash Commands`
2. Use the generated URL to install the bot to your server (or DM)

## 4. Connect Discord to clemvion

**GUI flow (recommended)**:

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/triggers/page.tsx"
  symbol="createMutation"
  describes="Chat Channel section in the trigger create dialog — Discord provider with Bot Token and Application Public Key"
/>

1. Sidebar → [**Triggers**](/triggers) → top-right **"+ Add Webhook Trigger"**
2. Enter trigger name, target workflow, and auth type
3. Toggle **"Connect a Chat Channel (external bot)"** at the bottom of the form
4. **Provider** dropdown = `Discord`
5. **Bot Token** = the token from Discord Developer Portal **Bot** tab → **Reset Token**
6. **Application Public Key** = the 64-char hex from Developer Portal **General Information**
7. Click **Create** — the backend:
   - calls `GET /applications/@me` to cache application id/name
   - bulk-overwrites slash commands via `PUT /applications/{app_id}/commands` (`start` / `cancel` / `help` / `reply`)
8. Copy the generated endpoint URL and paste it into **Interactions Endpoint URL** on the Discord Developer Portal **General Information** page
   - Discord sends a PING (`type: 1`) immediately; clemvion responds automatically with `{ type: 1 }` 200 — validation passes

<Callout type="note">
Copy the endpoint URL using the 📋 button on the trigger list. Slash command availability may take up to 1 hour after bulk-overwrite due to Discord cache propagation.
</Callout>

## 5. Usage

### 5.1 Slash commands

- `/workflow start` — new execution
- `/workflow cancel` — cancel active
- `/workflow help` — help
- `/workflow reply <message>` — reply during active AI Multi Turn

### 5.2 Buttons / Modals

- Button Presentation nodes → Discord Message Components (`ACTION_ROW` + `BUTTON`)
- A "Reply" button under AI messages opens a TEXT_INPUT modal (alternative to slash command)

### 5.3 Forms

- v1 = multi-step text sequence (one question per field, answered via buttons/modal)
- v2 (planned) = Discord modal (TEXT_INPUT components) native support

### 5.4 AI Agent render tools / Template

- When an AI Multi Turn node calls a `render_carousel` / `render_table` / `render_chart` / `render_template` tool, the resulting card / table / chart / template is sent as a sequential message right after the AI's text reply (`render_form` is tracked in a separate plan)
- Template nodes (non-blocking case without buttons) also send `output.rendered` as a text message automatically
- v1 fallback policy — chart/table render as monospace text, carousel as per-card sequential messages, template as plain text. SSR PNG upgrade is tracked in plan `chat-channel-visual-ssr-png`

## 6. v1 limitations / known gaps

- **No free-text DM** — v1 is Interactions Webhook only. Discord's `MESSAGE_CREATE` requires Gateway → v1 users reply via `/workflow reply` or modal (per CCH-MP-01 partial deferral, R-CC-13).
- **DM only** — guild (server) channels rejected. Multi-user channels bundled with v2 multi-user thread plans.
- **No auto-start** — unlike Telegram `/start` via first DM, Discord requires explicit `/workflow start`.
- **Form `file` field largely unsupported** — file upload needs Gateway attachment. Workaround: external storage URL (text + url validation).
- **Native typing** — `POST /channels/{id}/typing` (10s). Long LLM latency repeats every 5s.

## 7. Customizing notification messages (`languageHints` / `languageLocale`)

You can override the bot's built-in notification messages via `config.chatChannel.languageHints`.

### 7.1 `languageLocale` — default locale

`languageLocale` selects the **language of built-in default messages** for keys that have no override.

| Value | Description |
|---|---|
| `"ko"` | Korean defaults (**default** when omitted) |
| `"en"` | English defaults |

**Lookup order** (per key):
1. `languageHints[key]` — operator override always wins
2. Built-in default for the chosen `languageLocale`
3. `"ko"` fallback when `languageLocale` is also omitted

### 7.2 Execution failure keys (CCH-ERR-*)

When a workflow execution fails, the bot sends the user a failure-specific message. You can override any of the 6 keys below.

<Callout type="note">
Sensitive-data protection: the raw `error.message`, `nodeId`, `executionId`, and `details.url` are never included in notification messages. The only allowed placeholder in operator overrides is `{statusCode}`. Any other `{...}` placeholder returns 400 `VALIDATION_ERROR` (`details.code='UNKNOWN_PLACEHOLDER'`).
</Callout>

| Key | Sent when | `{statusCode}` allowed |
|---|---|---|
| `executionFailedThirdParty4xx` | External service (LLM / MCP / HTTP node) rejected with 4xx (`HTTP_4XX`) | optional |
| `executionFailedThirdParty5xx` | External service 5xx transient error (`HTTP_5XX`) | optional |
| `executionFailedThirdParty` | External service unreachable (no status code, not a timeout) | no |
| `executionFailedTimeout` | Processing timed out (`HTTP_TIMEOUT` / `LLM_TIMEOUT` / `EXECUTION_TIMEOUT` etc.) | no |
| `executionFailedRateLimit` | Rate limit exceeded (`LLM_RATE_LIMIT`) | no |
| `executionFailedInternal` | Server-side error (DB / CODE / SUB_WORKFLOW failure + unknown fallback) | no |

**EN default messages** (`languageLocale: "en"`):

```
executionFailedThirdParty4xx  → The external service rejected the request ({statusCode}). Please try again later.
executionFailedThirdParty5xx  → The external service is temporarily unavailable ({statusCode}). Please try again later.
executionFailedThirdParty     → Couldn't reach the external service. Please try again later.
executionFailedTimeout        → The request timed out. Please try again later.
executionFailedRateLimit      → Too many requests. Please try again later.
executionFailedInternal       → The service is temporarily unavailable. Please try again later.
```

## 8. Troubleshooting

| Symptom | Cause / fix |
|---|---|
| Discord Portal "Validation Failed" | Public Key mismatch or endpoint can't respond `{ type: 1 }` 200. Check `chat_channel_health` |
| Slash commands not visible | bulk-overwrite cache propagation ≤ 1h; or missing `applications.commands` scope |
| `401 Unauthorized` | Bot Token rotated — use **Rotate Bot Token** in clemvion (24h grace) |
| Modal "Reply" doesn't open | 3-second ack missed — backend latency (degraded health) |
