This page explains how to register and manage model configurations in the three tabs of the [Model Settings](/models) screen — **Chat**, **Embedding**, and **Rerank**. Each tab maintains its own independent list, and AI nodes and Knowledge Bases reference the configurations in each tab.

## Chat tab — LLM provider setup

The **Chat** tab manages the LLM providers used by AI Agent, Text Classifier, Information Extractor, and other AI nodes.

### Supported providers

### Adding a Chat model config

**GUI flow**: Click [Model Settings](/models?tab=chat) in the left sidebar and select the **Chat** tab. Click `[+ Add Model]` at the top right.

<Steps>
<Step>Pick a provider type from the dropdown: OpenAI · Anthropic · Google AI · Azure OpenAI · Local.</Step>
<Step>Enter an alias and the API Key. For Azure / Local (custom endpoints), also enter a Base URL.</Step>
<Step>Click `Load models` to fetch the provider's model list, then select the default model from the list. The form can fetch the list before saving using the credentials you entered.</Step>
<Step>Click `Test Connection` to verify a lightweight call (e.g. list-models) succeeds.</Step>
<Step>Save. Mark your most-used provider as `⭐ Default` so newly-created AI nodes auto-select it.</Step>
</Steps>

<Callout type="warn">API Keys are encrypted with AES-256-GCM. API responses return only masked previews like `sk-...xxxx`, so keep the original key elsewhere (usually the provider console).</Callout>

## Embedding tab — Embedding model setup

The **Embedding** tab manages the embedding models used by Knowledge Bases to convert documents into vectors. You select an embedding config from the embedding model dropdown when creating or editing a Knowledge Base.

### Supported providers

Embedding configs share the same provider structure as Chat configs (OpenAI · Google AI · Azure · Local). Anthropic does not support embeddings.

### Adding an Embedding model config

**GUI flow**: Click [Model Settings](/models?tab=embedding) in the left sidebar and select the **Embedding** tab. Click `[+ Add Model]` at the top right.

  symbol="ModelsPage"
  describes="Unified Models screen — Embedding tab for adding and managing embedding model configs"
/>

<Steps>
<Step>Pick a provider that supports embeddings (OpenAI · Google AI · Azure · Local).</Step>
<Step>Enter an alias and the API Key (if required). For Local (Ollama/vLLM), also enter a Base URL.</Step>
<Step>Click `Load models` to fetch the provider's model list, then select the default model from the list.</Step>
<Step>Click `Test Connection`. When the connection succeeds, the dimension is auto-detected via a probe embed call and saved automatically. </Steps>

<Callout type="note">Once an embedding dimension is saved (either auto-detected or manually entered), the field is locked to read-only to prevent accidental changes. The dimension cannot be changed after a Knowledge Base is created using this config — re-embedding would be required.</Callout>

## Rerank tab — Reranker setup

The **Rerank** tab manages reranker providers used to improve retrieval precision in Knowledge Bases. After setup, you can activate reranking per collection from the [Knowledge Base](/docs/06-integrations-and-config/knowledge-base) Reranking tab.

### What is a reranker?

Vector similarity search is fast, but it can return chunks that are semantically similar in a mixed order. A **reranker (cross-encoder model)** rescores each `(query, chunk)` pair directly in a post-retrieval step, selecting the most relevant top results in a more accurate order.

Key characteristics:

- Applied only at query time — no re-embedding of existing documents is required.
- Falls back gracefully to the original cosine order when the reranker encounters an error.
- Can be enabled or disabled per Knowledge Base.

### Setting a default reranker

Click `⭐ Default` in the list to designate the workspace-level default reranker. Knowledge Bases that have reranking enabled but no explicit reranker selected will use this default. If no default exists, reranking is safely bypassed for those collections.

<Callout type="note">Changing the default takes effect on the next search. It does not affect Knowledge Bases that already have a reranker explicitly selected.</Callout>

### Where the reranker is used

Registered rerankers appear in the [Knowledge Base](/docs/06-integrations-and-config/knowledge-base) Reranking tab. Activating reranking on a collection means every AI Agent search against that collection automatically goes through the second-pass precision step.

## API integration reference

Model configs are managed via the `/api/model-configs` endpoint. All types (Chat · Embedding · Rerank) share a single endpoint, differentiated by the `kind` query parameter.

## Tips & notes

- Change the Chat default provider using the `⭐ Default` button in the list. Existing AI nodes aren't affected — only newly created ones.
- Anthropic doesn't support embeddings. Even if Anthropic is your Chat default, register a separate Embedding config in the Embedding tab for Knowledge Base use.
- If you run a TEI server with a GPU, `dragonkue/bge-reranker-v2-m3-ko` significantly improves Korean retrieval quality.
- Cohere is a cloud API — quick to get started without hosting infrastructure.
- Deleting a reranker that is currently used by a Knowledge Base causes that KB to fall back to the cosine-only path.
- For details on per-KB reranking options (candidate pool size, score cutoff), see [Knowledge Base](/docs/06-integrations-and-config/knowledge-base).
- How to tune per-node parameters: [AI nodes](/docs/02-nodes/ai).
- For Azure OpenAI, the model ID must be the deployment name. Using the standard model ID returns 404.
