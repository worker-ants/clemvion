이 페이지에서는 외부 서비스와의 통합(Integration, 연동)을 중앙에서 관리하는 방법을 설명해요. HTTP Request나 Send Email 같은 노드가 여기서 만든 연동을 재사용해요.

## 통합이란

통합은 외부 서비스에 접근하기 위한 자격 증명 묶음이에요. 한 번 만들어 두면 여러 워크플로우의 여러 노드가 같은 계정·키를 공유해서 쓸 수 있고, 키 회전이나 재인증도 한 곳에서 끝낼 수 있어요.

## 팀 워크스페이스에서 공유 사용

팀 워크스페이스에 만든 통합은 그 워크스페이스의 모든 멤버가 자동으로 함께 사용할 수 있어요. 사용자가 사이드바에서 워크스페이스를 전환하면 통합 목록도 해당 워크스페이스 기준으로 갱신돼요.

## 연동 추가하기

[통합 관리](/integrations) 목록의 우측 상단에서 `[+ Add Integration]` 을 눌러요. 추가 플로우는 목록 모달에서 서비스를 고르고, 인증 단계부터 전용 페이지(`/integrations/new?service=...&step=auth`)로 전환돼요.

<Steps>
<Step>서비스 선택 모달에서 Google, GitHub, HTTP, Database, Email, Webhook 중 하나를 클릭해요.</Step>
<Step>Step 2 화면에서 별칭(`name`)과 범위(Personal / Organization)를 정해요. 팀 워크스페이스의 Organization 범위는 Admin만 선택할 수 있어요.</Step>
<Step>서비스별 자격 증명을 입력해요. OAuth 서비스는 scope 체크박스를 고른 뒤 `[Connect with ...]` 을 눌러 팝업으로 인증해요.</Step>
<Step>Step 3에서 자동으로 `POST /api/integrations/preview-test` 가 실행되며, 성공해야 `[Save integration]` 버튼이 활성화돼요.</Step>
<Step>저장에 성공하면 상세 페이지(`/integrations/[id]`)로 이동하고 `Integration created` 토스트가 떠요.</Step>
</Steps>

<Callout type="warn">OAuth 팝업이 5분 안에 돌아오지 않으면 타임아웃이 발생해요. 이때 입력한 자격 증명은 보안상 복원되지 않으니 Step 2부터 다시 시도해요.</Callout>

## 노드에서 연동 선택하기

워크플로우 에디터에서 HTTP Request, Send Email, Database Query 같은 노드를 추가하면, 설정 패널 상단에 `Integration` 드롭다운이 있어요. 드롭다운에는 해당 서비스 유형과 일치하는 연동만 나열돼요.

<Example title="HTTP Request 노드에서 연동 쓰기">
```ts
// 설정 패널에서 선택한 값은 노드 config에 저장돼요.
{
  "type": "httpRequest",
  "config": {
    "integrationId": "int_01H...",   // 통합 관리에서 만든 연동 ID
    "method": "POST",
    "path": "/v1/orders",            // base_url 뒤에 붙는 상대 경로
    "body": "{{ $run.input }}"
  }
}
```
</Example>

Send Email 노드도 같은 방식이에요. SMTP 연동의 `default_from` 값이 자동으로 기본 발신자가 되고, 필요하면 노드에서 개별적으로 덮어쓸 수 있어요.

## 팁 & 참고

- 자격 증명은 AES-256-GCM으로 암호화돼 저장되고, API 응답에서는 `****4f2a` 처럼 마스킹된 프리뷰만 돌아와요.
- 비OAuth 연동의 `Rotate credentials` 는 새 값의 테스트가 통과해야 기존 값을 덮어써요. 실패하면 기존 자격 증명은 그대로 유지돼요.
- 하나의 서비스에 여러 계정을 등록하려면 별칭을 다르게 해서 반복 추가하면 돼요. 워크스페이스 내에서 별칭은 유일해야 해요.
- 사용처가 있는 연동은 삭제할 수 없어요. 먼저 노드의 `integrationId` 를 다른 연동으로 바꾸거나, 해당 노드를 제거해요.
- AI Agent에서 외부 도구·리소스를 LLM에게 노출하려면 MCP 서버 통합을 등록하고 AI Agent 노드의 `MCP Servers` 필드에 연결해요. 자세한 흐름은 [MCP 서버 통합](/docs/06-integrations-and-config/mcp-servers)에서 확인해요.
