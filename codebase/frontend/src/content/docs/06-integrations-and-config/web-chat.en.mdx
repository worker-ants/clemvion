This page explains how to create and manage embedded web-chat widgets from within the product, set up the appearance, and embed the widget on your website.

**Web Chat** lets you add an AI chat widget to your website with a single install script, connected to a workflow you choose. You can customize the widget's color, name, welcome message, and more — then verify everything with a live preview right in the console.

<Callout type="note">
If you want to integrate the widget directly using the CDN snippet or npm package, or build your own UI (BYO-UI), see [Web Chat widget direct integration (SDK)](/docs/06-integrations-and-config/web-chat-sdk).
</Callout>

Click **Web Chat** in the left sidebar to open the [Web Chat console](/web-chat). It sits below the Schedule menu item.

The console shows the list of web-chat instances in your workspace, and — for the selected instance — the appearance settings, install script, and live preview.

## 2. Creating a web chat

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"
  symbol="CreateWebChatDialog"
  describes="Create web chat dialog — workflow selection and name input"
/>

Click **+ Create web chat** in the top-right corner.

<Steps>
<li>Select the **workflow** to connect. When a conversation starts through this web chat, the workflow runs. Set up your workflow in [Workflows](/workflows) beforehand.</li>
<li>Enter a **name**. This label identifies the instance in the console list.</li>
<li>Click **Create** to finish.</li>
</Steps>

<Callout type="note">
Creating a web chat and editing its appearance require the **editor** role or above. Viewing, copying the install script, and using the live preview are available to **viewers** and above.
</Callout>

## 3. Appearance and content settings

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"
  symbol="AppearanceBuilder"
  describes="Appearance/content builder — color, position, bot name, welcome message, suggestions, disclaimer"
/>

Select an instance from the list to reveal the **Appearance/Content** settings.

| Setting | Description |
|---|---|
| **Primary color** | Theme color for the chat button and header |
| **Position** | Bottom-right or bottom-left of the screen |
| **Bot name** | Name displayed at the top of the chat panel |
| **Welcome message** | Greeting shown immediately when the chat panel opens |
| **Suggested questions** | Question bubbles a user can tap to send as the first message. Enter one per line |
| **Disclaimer** | AI disclaimer text shown at the bottom of the chat |

These values are embedded directly in the install script. Click **Save** to persist them on the server — your settings are then preserved across browsers and team members. If you leave the page before saving, unsaved changes may be lost (a notice appears next to the Save button when there are unsaved changes).

<Callout type="tip">
Changes to the appearance settings are reflected instantly in the **Live preview** on the right. Adjust your settings while watching the preview, then click **Save** when you're happy.
</Callout>

## 4. Copying the install script

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"
  symbol="InstallSnippetBox"
  describes="Install script code box and copy button"
/>

In the **Install script** area below the appearance settings, click the **Copy** button. The full script is copied to your clipboard.

Paste the script at the end of the `<body>` tag in your website's HTML to activate the widget.

```html
<script>
  (function(d,s){var j=d.createElement(s);j.async=1;
   j.src="https://<widget-host>/web-chat/v1/loader.js";
   d.head.appendChild(j);})(document,"script");
</script>
<script>
  ClemvionChat('boot', {
    apiBase: 'https://<api-domain>',
    triggerEndpointPath: '...',
    locale: 'en',
    appearance: { primaryColor: '#5B4FE9', position: 'bottom-right' },
    headerTitle: 'Bot name',
    welcome: {
      text: 'Hi! How can I help you today?',
      suggestions: ['Question example 1', 'Question example 2']
    },
    disclaimer: 'AI responses may not always be accurate.'
  });
</script>
```

<Callout type="note">
The widget path in the script is not a secret. It is safe to leave it visible in your site source.
</Callout>

## 5. Live preview

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"

The **Live preview** appears to the right of the appearance settings. The actual widget boots there — the launcher button and chat panel render, and you can have a real conversation with the connected workflow to verify everything works.

The widget is included with the product, so no external setup is needed. Appearance changes appear in the preview right away.

## Tips & reference

- **Prepare your workflow first**: Create your [workflow](/workflows) before making a web chat — it makes selection easier.
- **Save then re-copy the script**: After updating the appearance settings, click **Save** first, then copy the script again and update your site's HTML. Changes are not synced automatically.
- **Run multiple web chats**: You can create several web chats for different purposes (customer support, pricing inquiries, etc.) and connect each to a different workflow. Use names to tell them apart in the list.
