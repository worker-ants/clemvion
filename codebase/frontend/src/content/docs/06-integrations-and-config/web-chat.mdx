이 페이지에서는 웹채팅 콘솔을 사용해 임베드 위젯을 만들고, 외형을 설정하고, 자기 웹사이트에 삽입하는 방법을 설명해요.

**웹채팅**은 워크플로우와 연결된 AI 채팅 위젯을 내 웹사이트에 한 줄의 스크립트로 추가할 수 있는 기능이에요. 콘솔에서 위젯의 색상·이름·환영 메시지 등 외형을 설정하고, 라이브 미리보기로 대화까지 바로 확인할 수 있어요.

<Callout type="note">
CDN 스니펫이나 npm 패키지로 위젯을 직접 통합하거나 자체 UI(BYO-UI)를 구현하려는 개발자는 [웹채팅 위젯 직접 통합 (SDK)](/docs/06-integrations-and-config/web-chat-sdk)을 참고해요.
</Callout>

## 1. 콘솔 진입

좌측 사이드바에서 **웹채팅**을 클릭하면 [웹채팅 콘솔](/web-chat)로 이동해요. 스케줄 메뉴 아래에 있어요.

콘솔에는 워크스페이스에 만들어진 웹채팅 인스턴스 목록과, 선택한 인스턴스의 외형 설정·설치 스크립트·라이브 미리보기가 표시돼요.

## 2. 웹채팅 만들기

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"
  symbol="CreateWebChatDialog"
  describes="웹채팅 만들기 다이얼로그 — 워크플로우 선택과 이름 입력"
/>

오른쪽 상단의 **+ 웹채팅 만들기** 버튼을 클릭해요.

<Steps>
<li>연결할 **워크플로우**를 선택해요. 웹채팅을 통해 대화가 시작되면 이 워크플로우가 실행돼요. [워크플로우 목록](/workflows)에서 미리 워크플로우를 준비해 두세요.</li>
<li>**이름**을 입력해요. 콘솔 목록에서 인스턴스를 구분하는 라벨이에요.</li>
<li>**만들기**를 클릭하면 웹채팅이 생성돼요.</li>
</Steps>

<Callout type="note">
웹채팅 만들기와 외형 편집은 **편집자(editor) 이상** 역할이 필요해요. 조회·설치 스크립트 복사·미리보기는 **뷰어(viewer) 이상**이면 가능해요.
</Callout>

## 3. 외형 및 콘텐츠 설정

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"
  symbol="AppearanceBuilder"
  describes="외형/콘텐츠 빌더 — 색상·위치·봇 이름·환영 메시지·추천 질문·고지 문구 편집 폼"
/>

목록에서 웹채팅 인스턴스를 선택하면 **외형/콘텐츠** 설정 영역이 나타나요.

| 설정 항목 | 설명 |
|---|---|
| **주요 색상** | 채팅 버튼과 헤더의 테마 색상 |
| **위치** | 화면 오른쪽 아래 또는 왼쪽 아래 중 선택 |
| **봇 이름** | 채팅 패널 상단에 표시되는 봇의 이름 |
| **환영 메시지** | 채팅 패널이 열릴 때 즉시 표시되는 인사 문구 |
| **추천 질문** | 사용자가 탭하면 첫 메시지로 전송되는 질문 버블. 줄바꿈으로 구분해 여러 개 입력 가능 |
| **고지 문구** | 채팅 하단에 표시되는 AI 면책 문구 |

설정 값은 설치 스크립트에 그대로 포함돼요. **저장** 버튼을 클릭하면 서버에 영속화되어, 브라우저나 운영자가 바뀌어도 동일한 설정이 유지돼요. 저장하지 않은 변경이 있으면 저장 버튼 옆에 안내 문구가 표시돼요.

<Callout type="tip">
설정을 바꾸면 오른쪽 **라이브 미리보기** 영역에 변경 사항이 바로 반영돼요. 미리보기를 보면서 원하는 외형을 맞춰 보고, 마음에 들면 **저장**을 눌러요.
</Callout>

## 4. 설치 스크립트 복사

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"
  symbol="InstallSnippetBox"
  describes="설치 스크립트 코드 박스와 복사 버튼"
/>

외형 설정 아래의 **설치 스크립트** 영역에서 **복사** 버튼을 클릭하면 스크립트 전체가 클립보드에 복사돼요.

복사한 스크립트를 자기 웹사이트 HTML의 `<body>` 끝에 붙여 넣으면 위젯이 활성화돼요.

```html
<script>
  (function(d,s){var j=d.createElement(s);j.async=1;
   j.src="https://<위젯 주소>/web-chat/v1/loader.js";
   d.head.appendChild(j);})(document,"script");
</script>
<script>
  ClemvionChat('boot', {
    apiBase: 'https://<API 도메인>',
    triggerEndpointPath: '...',
    locale: 'ko',
    appearance: { primaryColor: '#5B4FE9', position: 'bottom-right' },
    headerTitle: '봇 이름',
    welcome: {
      text: '안녕하세요! 무엇을 도와드릴까요?',
      suggestions: ['질문 예시 1', '질문 예시 2']
    },
    disclaimer: 'AI 답변은 부정확할 수 있어요.'
  });
</script>
```

<Callout type="note">
스크립트 안의 위젯 경로는 비밀 값이 아니에요. 사이트 소스에 그대로 노출되어도 괜찮아요.
</Callout>

## 5. 라이브 미리보기

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/web-chat/page.tsx"
  symbol="LivePreview"
  describes="콘솔 안에서 실제 위젯을 부팅해 대화까지 시연하는 라이브 미리보기 영역"
/>

외형 설정 영역 오른쪽에 **라이브 미리보기**가 표시돼요. 실제 위젯이 부팅되어 런처 버튼·채팅 패널이 렌더되고, 선택한 워크플로우와 실제 대화까지 시연할 수 있어요.

위젯은 제품과 함께 제공되므로 별도 외부 설정 없이 바로 동작해요. 외형 설정을 바꾸면 미리보기에 즉시 반영돼요.

## 팁 & 참고

- **워크플로우 먼저 준비하세요**: 웹채팅을 만들기 전에 [워크플로우](/workflows)를 미리 작성해 두면 선택이 편해요.
- **외형은 저장 후 스크립트 재복사로 적용**: 외형 설정을 바꾸고 **저장**한 뒤, 스크립트를 다시 복사해 HTML을 업데이트해야 해요. 자동 동기화는 되지 않아요.
- **여러 웹채팅 운영**: 목적(고객지원·가격문의 등)에 따라 여러 웹채팅을 만들어 각각 다른 워크플로우에 연결할 수 있어요. 목록에서 이름으로 구분해요.
