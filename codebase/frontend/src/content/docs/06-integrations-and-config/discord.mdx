Discord 봇을 워크플로우의 인터페이스로 연결하면, slash command (`/workflow start`) 와 button / modal 기반 인터랙션으로 워크플로우를 시작·조작할 수 있어요.

<Callout type="warn">
{/* 내부 SoT: spec/5-system/15-chat-channel.md R-CC-13 + spec/4-nodes/7-trigger/providers/discord.md R-D-3 */}
Discord v1 한계: 일반 DM 텍스트 메시지는 미지원이에요. v1 은 Interactions Webhook 방식으로 동작하며, Discord 의 자유 텍스트 DM 수신은 Gateway WebSocket 연결이 필요해 v1 범위 밖이에요. v1 사용자 reply 경로: (a) `/workflow reply <message>` slash command text option, (b) AI 응답 메시지의 **Reply** 버튼 클릭 후 modal TEXT_INPUT (기본 UX).
</Callout>

## 1. Discord 앱 만들기

1. [Discord Developer Portal](https://discord.com/developers/applications) → **New Application**
2. 생성된 앱의 **General Information** 페이지에서 다음 값 확인:
   - **Application ID** — 앱 식별자
   - **Public Key** — Interactions Webhook 의 ed25519 서명 검증 키 (64 chars hex)
3. **Bot** 탭 → **Reset Token** 으로 **Bot Token** 발급 (`xxx.yyy.zzz` 형식)

<Callout type="warn">
Bot Token 은 봇 계정을 완전히 제어하는 자격증명이에요. Public Key 는 공개 정보지만, 둘 다 clemvion 의 secret store 가 backend AES-256-GCM 으로 동일하게 관리해 일관성을 유지해요.
</Callout>

## 2. Interactions Endpoint URL 설정

Discord Developer Portal 의 앱 **General Information** 페이지 하단:

- **Interactions Endpoint URL** = `https://<your-clemvion-domain>/api/hooks/<endpointPath>`
- 저장 시 Discord 가 즉시 PING handshake (type=1) 전송 → clemvion 이 자동으로 `{ type: 1 }` 200 응답
- 응답 실패 시 Discord 가 "Validation Failed" 표시 → public key 또는 endpoint 도달 가능성 확인

## 3. Bot 권한 + Invite

1. **OAuth2 → URL Generator**
   - **scopes**: `bot`, `applications.commands`
   - **bot permissions**: `Send Messages`, `Read Message History`, `Use Slash Commands`
2. 생성된 URL 로 봇을 본인 server (또는 personal DM 활성) 에 install

## 4. clemvion 트리거에 Discord 연결

**GUI 등록 흐름 (권장)**:

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/triggers/page.tsx"
  symbol="createMutation"
  describes="트리거 생성 dialog 의 Chat Channel 섹션 — Discord provider 선택 및 Bot Token / Application Public Key 입력"
/>

1. 좌측 메뉴 → [**Triggers**](/triggers) → 우측 상단 **"+ Webhook 트리거 추가"** 클릭
2. 트리거 이름·연결할 워크플로우·인증 방식 입력
3. 폼 하단 **"Chat Channel 연결 (외부 챗봇)"** 토글을 켜기
4. **Provider** 드롭다운 = `Discord` 선택
5. **Bot Token** 입력 — Discord Developer Portal **Bot** 탭 → **Reset Token** 으로 발급한 값
6. **Application Public Key** 입력 — Developer Portal **General Information** 의 64자리 hex Public Key
7. **만들기** 클릭 → backend 가:
   - `GET /applications/@me` 호출로 application id / name 캐시
   - `PUT /applications/{app_id}/commands` 로 slash command 4종 (`start` / `cancel` / `help` / `reply`) bulk overwrite 등록
8. 생성된 트리거의 endpoint URL 을 복사해 Discord Developer Portal **General Information** 의 **Interactions Endpoint URL** 에 입력
   - 저장 시 Discord 가 PING (`type: 1`) 을 보내고 clemvion 이 자동으로 `{ type: 1 }` 200 응답 → 검증 통과

<Callout type="note">
endpoint URL 복사는 트리거 목록 화면의 📋 버튼으로 가능해요. slash command 는 bulk overwrite 후 Discord 캐시 반영까지 최대 1시간 소요될 수 있어요.
</Callout>

## 5. 사용 방법

### 5.1 Slash command

- `/workflow start` — 새 execution 시작
- `/workflow cancel` — 진행 중 execution 취소
- `/workflow help` — 도움말
- `/workflow reply <message>` — AI Multi Turn 진행 중 자유 텍스트 reply

### 5.2 Button / Modal

- Button Presentation 노드 → Discord Message Components (`ACTION_ROW` + `BUTTON`)
- AI 응답 메시지 끝의 **Reply** 버튼 클릭 → modal TEXT_INPUT 으로 자유 텍스트 입력 (`/workflow reply` 의 대안)

### 5.3 Form

- 현재는 다단계 텍스트 시퀀스로 진행돼요 (필드별 한 줄 질문 + button/modal 응답)

### 5.4 AI Agent render 도구 / Template

- AI Multi Turn 노드가 `render_carousel` / `render_table` / `render_chart` / `render_template` 도구를 호출하면 응답 텍스트 다음에 카드·표·차트·템플릿이 sequential 메시지로 함께 발송돼요
- Template 노드 (버튼 없는 non-blocking 케이스) 의 `output.rendered` 본문도 텍스트로 자동 발송돼요
- 현재 표현 방식 — chart/table 은 monospace 텍스트, carousel 은 카드별 sequential 메시지, template 은 plain text

## 6. 현재 한계 / 알려진 사항

- **자유 텍스트 DM 미지원** — Discord 의 일반 `MESSAGE_CREATE` event 는 Gateway WebSocket 연결이 필요해요. 자유 채팅 대신 `/workflow reply` slash command 또는 modal 을 사용해 주세요.
- **DM only** — guild (server) 채널은 지원하지 않아요 (자동 거부).
- **자동 start 미지원** — Telegram `/start` 의 DM 첫 메시지 자동 시작과 달리, Discord 는 `/workflow start` 를 명시적으로 입력해야 해요.
- **Form `file` 필드 사실상 미지원** — file upload 가 Gateway attachment 기반이라 사용할 수 없어요. 외부 storage URL 입력 (`text` + url validation) 을 우회로 사용해요.
- **typing 은 native** — `POST /channels/{id}/typing` (10초 유지). LLM latency 가 길면 5초마다 반복 호출돼요.

## 7. 안내 메시지 커스터마이즈 (`languageHints` / `languageLocale`)

봇이 사용자에게 보내는 자체 안내 문구는 `config.chatChannel.languageHints` 로 override 할 수 있어요.

### 7.1 `languageLocale` — 기본 locale 선택

`languageLocale` 필드로 override 하지 않은 키의 **default 문구 언어**를 선택해요.

| 값 | 설명 |
|---|---|
| `"ko"` | 한국어 default 문구 사용 (**default**, 생략 시 동일) |
| `"en"` | 영어 default 문구 사용 |

**lookup 우선순위** (키 하나당):
1. `languageHints[key]` — 운영자 override 가 있으면 항상 우선
2. `languageLocale` 의 언어별 내장 default
3. `"ko"` fallback — `languageLocale` 도 생략됐을 때

### 7.2 실행 실패 안내 키 (CCH-ERR-*)

워크플로우 execution 이 실패하면 봇이 사용자에게 실패 유형에 맞는 안내 메시지를 발송해요. 아래 6개 키로 문구를 override 할 수 있어요.

<Callout type="note">
민감정보 보호: `error.message` 원문 · `nodeId` · `executionId` · `details.url` 등은 안내 메시지에 절대 노출되지 않아요. 운영자 override 에서 허용되는 placeholder 는 `{statusCode}` 1종뿐이에요. 다른 `{...}` placeholder 사용 시 400 `VALIDATION_ERROR` (`details.code='UNKNOWN_PLACEHOLDER'`) 가 반환돼요.
</Callout>

| 키 | 발송 조건 | `{statusCode}` 허용 |
|---|---|---|
| `executionFailedThirdParty4xx` | 외부 서비스(LLM / MCP / HTTP 노드)가 4xx 거부 (`HTTP_4XX`) | 선택 |
| `executionFailedThirdParty5xx` | 외부 서비스 5xx 일시 오류 (`HTTP_5XX`) | 선택 |
| `executionFailedThirdParty` | 외부 서비스 응답 없음 (timeout 제외, status code 없음) | 불허 |
| `executionFailedTimeout` | 처리 시간 초과 (`HTTP_TIMEOUT` / `LLM_TIMEOUT` / `EXECUTION_TIMEOUT` 등) | 불허 |
| `executionFailedRateLimit` | 요청량 초과 (`LLM_RATE_LIMIT`) | 불허 |
| `executionFailedInternal` | 서버 내부 오류 (DB / CODE / SUB_WORKFLOW 실패 + 미분류 fallback) | 불허 |

**KO default 문구** (`languageLocale: "ko"` 또는 생략 시):

```
executionFailedThirdParty4xx  → 외부 서비스 요청이 거부되었습니다 ({statusCode}). 잠시 후 다시 시도해 주세요.
executionFailedThirdParty5xx  → 외부 서비스에 일시적인 문제가 발생했습니다 ({statusCode}). 잠시 후 다시 시도해 주세요.
executionFailedThirdParty     → 외부 서비스 응답을 받지 못했습니다. 잠시 후 다시 시도해 주세요.
executionFailedTimeout        → 처리 시간이 초과되었습니다. 잠시 후 다시 시도해 주세요.
executionFailedRateLimit      → 요청량이 많아 잠시 후 다시 시도해 주세요.
executionFailedInternal       → 서비스에 일시적 문제가 발생했습니다. 잠시 후 다시 시도해 주세요.
```

## 8. 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| Discord Portal "Interactions Endpoint URL — Validation Failed" | Public Key 불일치 또는 endpoint 가 PING 에 `{ type: 1 }` 200 응답 못 함. clemvion 의 `chat_channel_health` 확인 |
| slash command 가 Discord 에서 안 보임 | bulk overwrite 후 캐시 반영까지 1시간 소요 가능. 또는 `applications.commands` scope 누락 |
| `401 Unauthorized` | Bot Token 만료 — clemvion 의 **Bot Token 재발급** API 로 24h grace 회전 |
| modal "Reply" 가 click 후 안 뜸 | 3초 ack 시한 초과 — backend latency 점검 (degraded health) |
