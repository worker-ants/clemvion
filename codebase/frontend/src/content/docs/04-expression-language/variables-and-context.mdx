## `$input`: 직전 노드 결과

가장 자주 쓰는 참조예요. 현재 노드의 **입력 포트에 들어온 데이터**를 가리켜요.

<Example title="직전 노드의 email 필드 참조">
```ts
{{ $input.email }}
```
</Example>

<Example title="중첩 필드 접근">
```ts
{{ $input.user.profile.nickname }}
```
</Example>

<Callout type="warn">중간 경로에 `null`이 있으면 에러가 나요. 옵셔널 체이닝 `?.`을 쓰면 체인 중간에 `null`이 나와도 결과가 `null`로 short-circuit 돼요. 예: `{{ $input.user?.name }}`, `{{ $node["Fetch User"]?.output?.profile?.age }}`. 기본값이 필요하면 `{{ $input.user?.name || "unknown" }}` 또는 삼항 연산을 함께 써요.</Callout>

## `$node["이름"]`: 임의 노드 결과 참조

`$input`은 **직전** 노드만 가리키지만, `$node`는 워크플로우 안의 **임의 노드** 결과를 끌어올 수 있어요.

<Example title="라벨로 참조">
```ts
{{ $node["HTTP Request"].output.data.items[0] }}
```
</Example>

<Example title="UUID로 참조 (라벨 변경에 안정적)">
```ts
{{ $node["3f2a9b1c-...-uuid"].output.result }}
```
</Example>

<Callout type="note">워크플로우 안의 노드 라벨은 고유해야 해요. 실수로 중복되면 실행 순서에 따라 `Label#2`, `Label#3` 같은 접미사가 자동으로 붙어요.</Callout>

## `$var`: 워크플로우 변수

Variable Declaration 노드로 선언한 변수는 `$var`로 읽고, Variable Modification 노드로 갱신해요.

<Example title="카운터 변수 참조">
```ts
{{ $var.counter + 1 }}
```
</Example>

## `$now`: 시간 참조

`$now`는 실행 시점 기준으로 한 번 평가되고, 같은 실행 내에서는 동일한 값이 유지돼요. 항상 **UTC(`...Z`)** 기준 ISO 8601 문자열이에요.

<Example title="현재 ISO 타임스탬프">
```ts
{{ $now }}
```
</Example>

<Example title="오늘 날짜로 파일 이름 만들기 (UTC 기준)">
```ts
report-{{ formatDate($now, "YYYY-MM-DD") }}.csv
```
</Example>

<Callout type="note">예전에 제공되던 `$today` 변수는 항상 UTC 기준으로 산출돼 한국 시간대에서 자정 직전에 전날로 표시되는 문제가 있어 제거됐어요. 같은 결과는 `formatDate($now, "YYYY-MM-DD")`(UTC 기준) 또는 `today()`(서버 로컬 TZ 기준) 함수로 얻을 수 있어요.</Callout>

## `$env`: 환경 변수

셀프 호스팅 환경에서 워크스페이스 관리자가 허용한 키만 노출돼요. API 키, 외부 서비스 URL 등을 워크플로우에서 안전하게 참조할 때 써요.

<Example title="환경 변수 참조">
```ts
{{ $env.API_BASE_URL }}/users
```
</Example>

## `$thread`: Conversation Thread

워크플로우가 실행되는 동안 누적된 **사용자 입력 + AI Agent 응답 turn** 의 읽기 전용 뷰예요. AI Agent 노드의 `contextScope` 자동 주입과는 별개로, 어떤 표현식에서나 명시적으로 참조할 수 있어요.

<FieldTable rows={[
  { name: "turns", type: "Array", description: "ConversationTurn 배열의 readonly 스냅샷이에요. 각 turn 은 `source`, `role`, `nodeId`, `data` 등을 포함해요." },
  { name: "length", type: "Number", description: "누적된 turn 개수예요." },
  { name: "text", type: "String", description: "전체 thread 를 system_text 포맷으로 렌더한 결과예요. 첫 접근 시 메모이즈돼요." }
]} />

<Example title="누적 turn 개수로 분기">
```ts
{{ $thread.length > 5 }}
```
</Example>

<Example title="전체 thread 텍스트를 후속 노드 입력으로 첨부">
```ts
{{ $thread.text }}
```
</Example>

<Example title="첫 turn 의 form 데이터 필드 참조 (Presentation Form turn 일 때)">
```ts
{{ $thread.turns[0].data.email }}
```
</Example>

<Callout type="note">
`$thread.text` 는 한 노드의 실행 컨텍스트 안에서 처음 접근할 때 전체 thread 를 렌더하고 그 결과를 캐시해요. 즉, **같은 노드의 표현식 안에서 여러 번 써도 1번만 렌더**돼요. 다만 ForEach/Loop 처럼 iteration 마다 새 노드 컨텍스트가 만들어지는 곳에서는 매번 다시 렌더되므로, 루프 안에서 자주 참조해야 하면 미리 변수 노드에 담아 두는 편이 안전해요.
</Callout>

<Callout type="tip">
대부분의 경우 AI Agent 노드의 [Conversation Context 설정](/docs/02-nodes/ai#conversation-context)으로 자동 주입하는 편이 더 간단해요. 직접 `$thread` 를 참조하는 건 (1) AI Agent 가 아닌 노드(예: Transform/HTTP/Code)에서 thread 를 가공할 때, (2) 일부 turn만 골라 쓰고 싶을 때 유용해요.
</Callout>

## 팁 & 참고

- 자동완성은 마지막 실행 결과의 스키마를 기반으로 해요. 한 번도 실행하지 않은 워크플로우에서는 필드 제안이 제한되니 **먼저 한 번 실행**해 보는 걸 권장해요.
- 자동완성은 **현재 노드에서 접근 가능한 조상 노드와 해당 컨테이너 스코프 변수**만 보여줘요. 분기에서 반대쪽 가지의 노드나 다른 루프 내부 노드는 목록에서 제외돼요.
- 저장된 표현식이 접근 불가능한 노드를 참조하거나 컨테이너 밖에서 `$loop`·`$item`·`$itemIndex`를 쓰면 **주황색 경고**가 필드 하단에 표시돼요. 런타임에서 값이 비는 원인을 편집 시점에 미리 발견할 수 있어요.
- 실전 패턴은 [치트시트](/docs/04-expression-language/cheatsheet)에서 확인할 수 있어요.
- 실행 결과와 입력·출력 확인 방법은 [실행 결과](/docs/05-run-and-debug/run-results)에서 다뤄요.
