이 페이지에서는 비밀번호를 바꾸는 방법과, 변경 시 다른 기기의 세션이 어떻게 처리되는지 설명해요.

## 비밀번호 변경하기

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/w/[slug]/profile/change-password/page.tsx"
  symbol="ChangePasswordPage"
  describes="프로필 → 비밀번호 변경 페이지 진입점"
/>

[프로필 → 비밀번호 변경](/profile/change-password) 페이지에서 진행해요. 보안 설정(2FA · Passkey)은 [프로필 → 보안](/profile/security) 에서 따로 관리해요.

<Steps>
<li>

[프로필 → 비밀번호 변경](/profile/change-password) 으로 이동해요.

</li>
<li>

**현재 비밀번호** 를 입력해요. 입력한 비밀번호가 일치해야 다음 단계로 넘어갈 수 있어요.

</li>
<li>

**새 비밀번호** 를 입력해요. 아래 정책을 모두 충족해야 해요.

- 최소 8자
- 영문 소문자 · 영문 대문자 · 숫자 · 특수문자 중 **3종 이상** 포함

</li>
<li>

새 비밀번호를 한 번 더 입력해 확인한 뒤 **변경** 을 눌러요.

</li>
</Steps>

변경이 완료되면 현재 브라우저는 로그아웃 없이 그대로 유지되고, 프로필 페이지로 돌아와요.

## 다른 기기의 세션은 어떻게 되나요?

비밀번호 변경에 성공하면 **현재 사용 중인 기기를 제외한 다른 모든 기기·세션이 즉시 로그아웃** 돼요.

분실한 기기나 의심스러운 세션을 한 번에 끊고 싶을 때 비밀번호 변경을 활용할 수 있어요. 현재 브라우저의 로그인 상태는 유지되므로 변경 직후에도 계속 작업할 수 있어요.

<Callout type="note">
  세션 관리에서 개별 세션을 직접 종료하고 싶다면 [프로필 → 세션](/profile/sessions) 에서 확인할 수 있어요.
</Callout>

## 비밀번호가 없는 계정 (소셜 로그인 전용)

Google · GitHub 등 소셜 로그인(OAuth)으로만 가입한 계정은 비밀번호가 설정되어 있지 않아요. 이 경우 현재 비밀번호 확인 단계에서 실패하므로 비밀번호 변경을 진행할 수 없어요.

소셜 로그인 계정의 비밀번호를 직접 설정하는 기능은 현재 제공되지 않아요.

## 이메일 변경하기

<ImplAnchor
  kind="ui-entry"
  file="codebase/frontend/src/app/(main)/w/[slug]/profile/change-email/page.tsx"
  symbol="ChangeEmailPage"
  describes="프로필 → 이메일 변경 페이지 진입점"
/>

[프로필](/profile) 화면의 사용자 정보에서 이메일 옆 **변경하기** 를 누르면 [이메일 변경 페이지](/profile/change-email) 로 이동해요.

<Steps>
<li>

[프로필 → 이메일 변경](/profile/change-email) 으로 이동해요.

</li>
<li>

**새 이메일 주소** 를 입력해요.

</li>
<li>

본인 확인을 위해 **현재 비밀번호** 또는 **2FA 코드** 를 입력해요.

</li>
<li>

**확인 메일 보내기** 를 눌러요. 입력한 새 이메일 주소로 확인 메일이 발송돼요.

</li>
<li>

받은 확인 메일의 링크를 클릭하면 이메일 변경이 확정돼요. 확정은 현재 로그인된 세션에서만 처리돼요.

</li>
</Steps>

### 변경 확정 후 세션 처리

이메일 변경이 확정되면 **현재 사용 중인 기기를 제외한 다른 모든 기기·세션이 즉시 로그아웃** 돼요. 비밀번호 변경 시와 동일한 보안 정책이에요.

### 기존 이메일로 발송되는 안내 메일

변경 확정과 동시에 **이전 이메일 주소** 로 "이메일이 변경되었어요" 안내 메일이 발송돼요. 본인이 요청한 변경이 아니라면 안내 메일의 링크를 통해 비밀번호를 재설정해 계정을 보호할 수 있어요.

### 진행 중인 변경 취소 또는 재발송

확인 메일을 받지 못하거나 변경을 취소하고 싶다면 [이메일 변경 페이지](/profile/change-email) 에서 **재발송** 또는 **취소** 를 선택할 수 있어요.

<Callout type="note">
  확인 링크는 발송 후 **1시간** 동안만 유효해요. 시간이 지났다면 재발송 후 새 링크를 사용해요.
</Callout>

### 이메일을 변경할 수 없는 경우

소셜 로그인(OAuth)으로만 가입하고 **비밀번호도 2FA도 설정하지 않은 계정** 은 본인 확인 수단이 없어 이메일을 변경할 수 없어요. 이메일 변경을 원한다면 먼저 비밀번호를 설정하거나 2FA를 활성화한 뒤 진행해요.

<Callout type="tip">
  2FA 설정은 [프로필 → 보안](/profile/security) 에서 할 수 있어요.
</Callout>
