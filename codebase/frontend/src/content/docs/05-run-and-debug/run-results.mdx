이 페이지에서는 **실행 이력 조회**, **Run Results 드로어의 타임라인**, **노드별 입출력 확인**, **에러 메시지 해석** 방법을 설명해요.

## Run Results 드로어

워크플로우를 실행하면 에디터 하단에 Run Results 드로어가 자동으로 펼쳐져요. **2분할 레이아웃**으로, 왼쪽은 타임라인, 오른쪽은 선택한 노드의 상세 결과를 표시해요.

왼쪽 타임라인에는 **모든 실행된 노드**가 실행 순서대로 나열돼요.

## AI 노드 Multi Turn 결과

AI Agent / Information Extractor의 Multi Turn 실행은 타임라인에서 접힘/펼침 가능한 트리로 나와요. 펼치면 **사용자 메시지, AI 응답, Tool 호출**이 프리뷰 카드로 인라인 표시돼요.

- 대화 진행 중이면 인스펙터 하단에 메시지 입력 UI가 sticky로 고정돼요.
- 완료된 대화는 채팅 스레드처럼 읽을 수 있고, 항목을 클릭하면 상세 패널 탭 구성이 바뀌어요:

### 멀티턴 대화 중 오류 발생 시 재시도

멀티턴 AI Agent 노드가 실행 중 LLM 오류(Rate Limit, 서버 오류, 타임아웃 등)로 종료되더라도 **대화 내용은 보존**돼요. 대화 인스펙터의 해당 `system_error` 항목 오른쪽에 **[다시 시도]** 버튼이 나타나며, 클릭하면 마지막 사용자 메시지를 다시 LLM에 전송해요.

<Callout type="note">
재시도 가능 여부는 오류 종류에 따라 달라요.

- **재시도 가능**: Rate Limit(429), 서버 오류(5xx), 네트워크·타임아웃 — 대화가 보존되고 [다시 시도] 버튼이 나타나요.
- **재시도 불가**: 인증 오류(401/403) 등 — 설정 문제로 재시도해도 해결되지 않으므로 버튼이 나타나지 않아요. [LLM 설정](/docs/06-integrations-and-config/llm-config)에서 인증 정보를 점검해요.

또한 재시도는 **60분 이내 한 번**만 가능해요. 60분이 지났거나 이미 재시도를 한 번 완료했다면 워크플로우를 처음부터 다시 실행해야 해요.

재시도가 성공하면 AI 노드 다음에 연결된 노드(예: HTTP Request, 이메일 전송 등)가 일반 실행과 동일하게 이어서 실행돼요.
</Callout>

## 실행 이력 조회

한 번의 실행이 끝난 뒤에도 결과를 다시 볼 수 있어요.

<Steps>
  <Step>에디터 헤더의 더보기(`⋯`) → `실행 히스토리`를 클릭하거나, Run Results 드로어의 `View All Executions` 링크를 눌러요.</Step>
  <Step>`/workflows/:id/executions` 페이지에서 상태별 필터(All / Completed / Failed / Running / Cancelled / Waiting)로 좁혀요.</Step>
  <Step>행을 클릭하면 `/workflows/:id/executions/:executionId` 상세 페이지로 이동해요.</Step>
  <Step>상세 페이지에서 실행 요약 카드, 노드별 결과, 에러 메시지, `← Prev / Next →`로 인접 실행 탐색까지 할 수 있어요.</Step>
</Steps>

실행 이력 진입점은 Dashboard의 Recent Executions, Workflow List의 컨텍스트 메뉴(`⋯ → Execution History`)에서도 찾을 수 있어요.

## 노드별 입출력 확인 팁

- **Copy 버튼**: 상세 뷰의 Input/Output 탭에서 `Copy Input`, `Copy Output` 버튼으로 JSON을 복사할 수 있어요. 그대로 Run with Input에 붙여넣으면 재현 실험이 쉬워요.
- **이 입력으로 다시 실행**: 실행 이력 상세에서 특정 실행의 입력으로 새 실행을 시작할 수 있어요.
- **Presentation 노드 Preview**: Table·Chart·Carousel·Template은 JSON이 아닌 렌더링된 프리뷰를 제공해요. 결과 JSON은 같은 화면의 Output 탭에 있어요.

## 팁 & 참고

- 실패를 견고하게 처리하려면 노드별로 [에러 처리](/docs/05-run-and-debug/error-handling) 정책을 설정해요.
- 실행 이전 상태로 되돌리려면 [버전 히스토리](/docs/05-run-and-debug/version-history)를 이용해요.
- 표현식 관련 에러는 [표현식 기초](/docs/04-expression-language/basics)와 [변수와 컨텍스트](/docs/04-expression-language/variables-and-context)에서 원인 단서를 찾을 수 있어요.
