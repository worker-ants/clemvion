이 페이지에서는 에디터에서 워크플로우를 **실행하는 세 가지 방식**과, 트리거 종류(Manual / Schedule / Webhook)에 따른 차이를 설명해요.

## 세 가지 실행 방식

에디터 상단 툴바의 `▶ Run` 버튼 드롭다운에서 선택할 수 있어요.

## Run with Input 다이얼로그

`Run with Input`을 선택하면 테스트 입력 다이얼로그가 열려요.

<Steps>
  <Step>JSON 에디터에 테스트 데이터를 직접 입력해요. 실시간으로 유효성 검증이 이루어져요.</Step>
  <Step>`Load from History` 드롭다운으로 이전 실행의 입력 데이터를 불러올 수 있어요.</Step>
  <Step>자주 쓰는 입력은 이름을 붙여 저장해두면 다음부터 원클릭으로 재사용할 수 있어요.</Step>
  <Step>`Run ▶`을 눌러 실행을 시작해요.</Step>
</Steps>

<Example title="Run with Input JSON 예시">
```json
{
  "userId": 123,
  "action": "signup",
  "data": {
    "email": "test@example.com"
  }
}
```
</Example>

## 테스트 데이터셋

자주 쓰는 테스트 입력을 **데이터셋**으로 저장해두면 다음 테스트부터 원클릭으로 재사용할 수 있어요.

### 데이터셋으로 저장

`Run with Input` 다이얼로그 우측 상단의 `저장` 버튼을 누르면 현재 JSON 입력을 데이터셋으로 저장해요.

<FieldTable rows={[
  { name: "이름", type: "필수", description: "데이터셋을 식별하는 이름. 같은 워크플로우 내 내 데이터셋과 중복될 수 없어요." },
  { name: "워크스페이스 공유", type: "선택 (기본: OFF)", description: "켜면 같은 워크스페이스 팀원이 읽기 전용으로 불러올 수 있어요. 공유본 수정이 필요하면 Clone 후 편집해요." }
]} />

### 데이터셋 불러오기

다이얼로그의 `Datasets` 드롭다운에서 저장된 데이터셋을 선택하면 JSON 에디터에 자동으로 채워져요.

- **내 데이터셋**: 내가 만든 데이터셋(private 포함)이 모두 표시돼요.
- **워크스페이스 공유본**: 팀원이 공유 설정한 데이터셋도 함께 표시돼요.

### 가시성(Visibility)

<FieldTable rows={[
  { name: "Private (기본)", type: "비공개", description: "나만 볼 수 있어요. 저장할 때 워크스페이스 공유를 켜지 않으면 이 상태예요." },
  { name: "Workspace", type: "워크스페이스 공유", description: "같은 워크스페이스 모든 Editor 이상 팀원이 읽기 전용으로 불러올 수 있어요." }
]} />

### 공유 데이터셋 복제 (Clone)

팀원이 공유한 데이터셋을 수정하고 싶을 때는 `Clone` 버튼을 눌러 내 private 사본을 만든 후 편집해요. 원본 공유 데이터셋은 변경되지 않아요.

<Callout type="note">Clone 된 사본의 이름은 자동으로 `원본 이름 (Copy)` 가 돼요. 같은 이름의 사본이 이미 있으면 409 오류가 반환돼요 — 이름을 바꿔서 다시 Clone 해요.</Callout>

### 데이터셋 삭제

`Delete` 버튼은 **내가 소유한 데이터셋**에만 표시돼요. 타인이 공유한 데이터셋은 삭제할 수 없어요.

## Run from Selected

디버깅 중인 노드 하나만 집중해서 확인하고 싶을 때 써요.

<Steps>
  <Step>캔버스에서 출발점이 될 노드를 선택해요.</Step>
  <Step>노드 우클릭 → `여기서부터 실행`을 선택해요.</Step>
  <Step>이전 실행에서 해당 노드가 받았던 입력이 자동으로 채워져요. 필요하면 수동으로 수정할 수 있어요.</Step>
  <Step>선택 노드부터 연결된 하류 노드까지만 실행돼요.</Step>
</Steps>

<Callout type="tip">AI Agent나 HTTP Request처럼 비용·시간이 드는 노드를 다시 돌리고 싶지 않을 때, 그 **다음 노드**부터 부분 실행하면 시간을 크게 아낄 수 있어요.</Callout>

캔버스에서 노드를 우클릭한 뒤 **이 노드 실행**을 선택하면 해당 노드 하나만 실행해요. 전체 실행이나 Run from Selected 와 달리 하류 노드는 실행되지 않아요.

- **입력**: 직전 실행이 있으면 그 실행에서 상류(직속 이전) 노드의 출력이 자동으로 입력으로 주입돼요. 직전 실행이 없으면 빈 입력으로 실행돼요.
- **결과**: 설정 패널 **Info** 탭에 해당 노드의 출력·오류·소요 시간이 표시돼요. 하단 Run Results 드로어 타임라인에도 해당 노드가 나타나요.

<Callout type="tip">먼저 워크플로우를 한 번 전체 실행한 뒤, 수정하고 싶은 노드를 우클릭해 단독 재실행하며 디버깅하면 효율적이에요.</Callout>

<Callout type="note">form, buttons, AI 멀티턴처럼 사용자 응답을 대기하는 대화형 노드와 Loop·ForEach·Map 같은 컨테이너 내부 노드는 단독 실행 대상이 아니에요. 표준·AI·코드·HTTP Request 등 일반 노드에서 사용할 수 있어요.</Callout>

## 트리거 종류와 실행 방식의 차이

**트리거**는 워크플로우가 실행되는 **시작 조건**이에요. 에디터의 Run 버튼은 어떤 트리거든 관계없이 수동으로 실행을 걸지만, 배포 이후에는 트리거별로 자동 실행돼요.

<FieldTable rows={[
  { name: "Manual Trigger", type: "수동", description: "사용자가 Run 버튼을 누를 때만 실행돼요. 테스트·디버깅·일회성 작업에 적합해요. `$execution.mode` 값은 `manual`이에요." },
  { name: "Schedule", type: "스케줄", description: "Cron 표현식 기반으로 주기적으로 실행돼요. 배치 작업, 정기 리포트, 데이터 수집에 써요. `$execution.mode` 값은 `schedule`이에요." },
  { name: "Webhook", type: "웹훅", description: "외부 시스템이 HTTPS로 호출하면 실행돼요. 페이로드는 `$trigger.body`로 들어와요. 이벤트 기반 자동화에 써요. `$execution.mode` 값은 `webhook`이에요." }
]} />

<Callout type="note">에디터에서 `Run`을 누르면 트리거 종류와 상관없이 항상 `manual` 모드로 실행돼요. Webhook 페이로드 시뮬레이션이 필요하다면 `Run with Input`에서 JSON을 직접 넣어요.</Callout>

## 실행 상태 확인

실행을 시작하면 에디터 상단에 실행 상태 바가 표시돼요.

```
▶ Running...   0.5s elapsed    [■ Stop]
```

- 현재 실행 중인 노드: 파란 펄스 테두리
- 완료된 노드: 초록 체크 + 소요 시간
- 연결선: 데이터 흐름 애니메이션
- 분기 노드: 실제 실행된 경로만 진하게, 미실행 경로는 흐릿하게

하단의 **Run Results 드로어**가 자동으로 펼쳐지며, 각 노드의 입력·출력·상태가 타임라인으로 나타나요. 자세한 내용은 [실행 결과](/docs/05-run-and-debug/run-results)에서 다뤄요.
