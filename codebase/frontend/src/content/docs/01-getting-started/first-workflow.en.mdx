This page walks through the most basic workflow you can build — **Manual Trigger + HTTP Request** — end to end. You'll call a public API once and see the result in Run Results.

## What you'll build

  { title: "Manual Trigger", sub: "Start signal" },
  { title: "HTTP Request", sub: "GET public API" },
]} />

When you run it, the HTTP Request node calls an external API and returns the response as the **node's result**.

## Before you start

- A signed-in user account.
- A public API URL you can call without setting up an Integration. We'll use `https://jsonplaceholder.typicode.com/todos/1` as an example.

## Step by step

<Steps>
  <Step>
    **Create a new workflow** — From the sidebar, open `Workflows` and click `+ New Workflow` in the top right. An empty canvas opens with a `Manual Trigger` node already placed in the middle. (Each workflow has exactly one Manual Trigger and you can't delete it.)
  </Step>

  <Step>
    **Name the workflow** — Click the name in the header and rename it to something like `My First Workflow`. The canvas auto-saves, but `Ctrl+S` saves on demand any time.
  </Step>

  <Step>
    **Add an HTTP Request node** — In the search box of the left palette, type `HTTP` and drag the `HTTP Request` node onto the canvas. Dropping it to the right of the Manual Trigger looks natural.
  </Step>

  <Step>
    **Connect the nodes** — Click-and-drag from the output port (a small circle on the right of Manual Trigger) to the input port on the left of HTTP Request. When the line appears, the data flow is wired up.
  </Step>

  <Step>
    **Fill in the HTTP Request settings** — Click the HTTP Request node to open the settings panel on the right, and set these values.

    <FieldTable rows={[
      {name: "method", required: true, type: "enum", description: "Choose GET.", default: "GET"},
      {name: "url", required: true, type: "string (expression)", description: "Enter `https://jsonplaceholder.typicode.com/todos/1`.", default: "-"},
      {name: "authentication", required: true, type: "enum", description: "Leave it as `none`. For APIs that require auth, create an Integration first.", default: "none"},
      {name: "responseType", required: false, type: "enum", description: "`json` parses the response into an object for you.", default: "json"}
    ]} />

    Settings apply instantly. When the `⚠ Not configured` badge disappears, you're ready.
  </Step>

  <Step>
    **Run it** — Click `▶ Run` in the top right of the header. Each node on the canvas changes color based on its execution state, and the **Run Results drawer** opens at the bottom. Manual Trigger finishes almost immediately; HTTP Request turns green when it succeeds.
  </Step>

  <Step>
    **Inspect the result** — In Run Results, click the HTTP Request node to see the `Preview / Input / Output / Error` sub-tabs. The `Output` tab shows something like this:

    <Example title="Output example">
    ```json
    {
      "data": {
        "userId": 1,
        "id": 1,
        "title": "delectus aut autem",
        "completed": false
      },
      "meta": {
        "statusCode": 200,
        "duration_ms": 120
      }
    }
    ```
    </Example>

    To use this result in the next node, reference it with an expression like `{{ $input.data.title }}`.
  </Step>
</Steps>

## Checklist when things go wrong

<Callout type="warn" title="If you hit an error">
- URL typos are the most common culprit. Make sure the `https://` prefix is there.
- If external APIs are blocked on your corporate network, the run goes out through the `error` port. Check the status code and message in the `Error` tab of Run Results.
- For APIs that require authentication, create the connection in `Integration` first and select it from the node settings.
</Callout>

<Callout type="tip">
Declare **input parameters** on the Manual Trigger and the run dialog will prompt you for values when you start the workflow. Downstream nodes can read them with `{{ $params.<name> }}` or `{{ $input.parameters.<name> }}`.
</Callout>

## Tips & notes

- To re-run just one node, right-click the node and choose `Execute` (single-node test) or `Run from here`. Details: [Running a workflow](/docs/05-run-and-debug/running-a-workflow).
- The layout of result tabs and the preview priority rules are documented in [Viewing run results](/docs/05-run-and-debug/run-results).
- Power-user tip: the HTTP Request node has two output ports — `success` and `error`. Wire the `error` port to a separate branch to handle failure paths. Field details: [Node guide · Integrations](/docs/02-nodes/integrations).
