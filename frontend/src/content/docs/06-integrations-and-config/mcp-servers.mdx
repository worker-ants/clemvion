이 페이지에서는 외부 **MCP(Model Context Protocol) 서버**를 워크스페이스 통합으로 등록하고, AI Agent 노드에 연결해 LLM이 그 서버의 도구·리소스·프롬프트를 직접 호출하도록 만드는 방법을 설명해요.

## MCP가 뭐고 언제 써요

MCP는 LLM이 외부 시스템의 **도구(tools)** · **리소스(resources)** · **프롬프트(prompts)** 를 표준 방식으로 호출할 수 있도록 정의한 프로토콜이에요. 워크플로우 안에 노드를 끌어다 놓는 일반 도구(Tool Area)와의 차이는 **도구 자체가 외부 시스템에 있다**는 점이에요.

언제 적합한가요.

- 사내 위키·검색 엔진처럼 **외부에서 운영되는 도구 모음**을 한 번에 LLM에게 노출하고 싶을 때.
- Slack 검색, GitHub 이슈 조회, 사내 결재 시스템 같은 도구를 **여러 워크플로우의 AI Agent에서 재사용**하고 싶을 때.
- 외부 팀이 운영하는 도구 카탈로그를 그쪽에서 갱신하면 **별도 코드 수정 없이** AI Agent가 새 도구를 인식하게 하고 싶을 때.

## MCP 서버 등록

MCP 서버는 다른 통합과 동일하게 `/integrations` 에서 등록해요. service type만 `MCP Server` 로 고르면 돼요.

<Steps>
  <Step>`/integrations` 우측 상단의 `[+ Add Integration]` 을 누르고 service-picker에서 **MCP Server** 를 선택해요.</Step>
  <Step>Step 2에서 별칭(`name`)과 워크스페이스 범위를 정해요. 팀 워크스페이스의 Organization 범위는 Admin만 선택할 수 있어요.</Step>
  <Step>Step 3에서 인증 방식을 골라 자격 증명을 입력해요. `Bearer Token` / `API Key` / `None` 중 하나예요.</Step>
  <Step>저장 직전 자동으로 `POST /api/integrations/preview-test` 가 실행돼요. 서버에 연결해 `tools/list` 응답과 `serverInfo`(이름·버전·capabilities)를 받아 보여줘요. 통과해야 `[Save integration]` 이 활성화돼요.</Step>
  <Step>저장에 성공하면 상세 페이지(`/integrations/[id]`)로 이동해요. Overview 탭에서 서버가 노출하는 도구 목록과 capabilities(resources·prompts 지원 여부)를 확인할 수 있어요.</Step>
</Steps>

<Callout type="warn">
연결 테스트(`preview-test`)에 실패하면 저장이 막혀요. 로그 메시지는 민감 정보 누출을 막기 위해 서버 응답이 일부 가공돼서 표시돼요. 자세한 원인은 서버 쪽 로그에서 확인하는 게 빨라요.
</Callout>

## AI Agent 노드에 연결하기

AI Agent 설정 패널의 **MCP Servers** 멀티 선택 필드에서 등록한 MCP 서버를 골라요. 항목별로 노출 정책을 세부 조정할 수 있어요.

<FieldTable rows={[
  { name: "integrationId", type: "필수", required: "필수", description: "등록한 MCP 통합의 ID예요. 한 노드 안에서 같은 `integrationId` 를 두 번 쓰면 저장이 거부돼요(서버 sid 충돌 방지)." },
  { name: "enabledTools", type: "string[]", required: "선택", description: "비워 두면 서버가 노출한 **모든 일반 도구**를 LLM에게 열어줘요. 일부만 쓰고 싶으면 도구 이름을 배열로 지정해요." },
  { name: "includeResources", type: "boolean", required: "선택", description: "기본 `true`. 서버가 resources capability를 보고하면 `list_resources` · `read_resource` 메타 도구를 LLM에게 노출해요." },
  { name: "includePrompts", type: "boolean", required: "선택", description: "기본 `true`. 서버가 prompts capability를 보고하면 `list_prompts` · `get_prompt` 메타 도구를 노출해요." },
  { name: "toolOverrides", type: "{ toolName, description? }[]", required: "선택", description: "특정 도구의 LLM 노출 설명을 덮어써요. 자주 헷갈리는 도구의 의도를 분명히 적어 두면 LLM 호출 정확도가 올라가요." }
]} />

LLM이 도구를 호출하면 AI Agent가 통합의 자격 증명을 사용해 MCP 서버에 요청하고, 결과를 다음 LLM 호출의 `tool_result` 로 다시 넣어줘요. 도구 이름은 LLM 측에서 충돌이 없도록 `mcp_<sid>__<toolName>` 형식으로 sanitize 되어 노출돼요.

<Example title="AI Agent에 MCP 서버 두 개 연결">
```ts
// AI Agent config 일부
{
  mcpServers: [
    {
      integrationId: "int_mcp_internal_search",
      enabledTools: ["search_wiki", "search_jira"],
      includeResources: true,
      includePrompts: false
    },
    {
      integrationId: "int_mcp_slack",
      // enabledTools 미지정 → 서버의 모든 일반 도구 노출
      toolOverrides: [
        { toolName: "post_message", description: "팀 채널에 공지를 보낼 때만 사용해. 1:1 DM에는 쓰지 마." }
      ]
    }
  ]
}
```
</Example>

## 재인증·도구 변경 시 동작

- 서버에서 도구가 사라지거나 추가되면 **다음 실행에서 자동으로 갱신**돼요. AI Agent가 매번 `tools/list` 를 다시 가져와 LLM에게 노출하는 도구 카탈로그를 새로 만들어요.
- 인증이 만료되거나 서버에 도달할 수 없으면 노드가 `error` 포트로 라우팅돼요. 통합 상세 페이지에서 자격 증명을 회전(`Rotate credentials`)하거나 재인증해요.
- Multi Turn 모드에서는 **첫 턴의 `mcpServers` 스냅샷이 이후 턴에 그대로 유지**돼요. 대화 도중 통합 설정을 바꿔도 같은 대화에서는 영향이 없고, 새 실행부터 반영돼요.

## 제약 사항

- **동일 노드 안에서 같은 `integrationId` 중복 금지** — 서버 sid 충돌이 나면 LLM이 도구를 구분하지 못해요. 검증에서 자동으로 거부돼요.
- 도구 응답 크기는 기본 **100KB**로 잘려요(`MCP_MAX_RESPONSE_BYTES`). 큰 응답이 잦으면 환경변수로 늘리거나 도구 쪽에서 페이지네이션을 적용해요.
- 단일 도구 호출은 기본 **30초**(`MCP_CALL_TIMEOUT_MS`), `tools/list` 는 10초, 연결 시도는 10초가 상한이에요.
- 인증 방식은 `bearer_token` · `api_key` · `none` 셋만 허용해요. 그 외 값은 등록 단계에서 차단돼요.

## 팁 & 참고

- LLM이 잘못된 도구를 고르는 게 잦다면 `enabledTools` 로 도구를 좁히거나, `toolOverrides.description` 으로 의도를 명확히 적어줘요.
- `includeResources` · `includePrompts` 가 켜져 있어도 서버가 해당 capability를 보고하지 않으면 메타 도구는 노출되지 않아요. 통합 상세의 capabilities 패널에서 서버 측 지원 여부를 확인해요.
- 워크플로우 내부의 다른 노드를 LLM 도구로 노출하고 싶다면 캔버스의 **Tool Area**를 사용해요. 외부 시스템 도구는 MCP, 내부 노드는 Tool Area로 역할이 분리돼 있어요.
- AI Agent의 조건 분기·KB·도구 일반 동작은 [AI 노드](/docs/02-nodes/ai)를 참고해요.
- 통합 관리 화면 흐름과 권한 정책은 [통합 관리](/docs/06-integrations-and-config/integration-management)에 정리되어 있어요.
