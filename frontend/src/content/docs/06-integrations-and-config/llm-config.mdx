이 페이지에서는 AI Agent, Text Classifier, Information Extractor 같은 AI 노드가 호출할 LLM 프로바이더를 등록하고 기본 모델·파라미터를 지정하는 방법을 설명해요. 지식 저장소의 임베딩 모델도 여기서 고른 프로바이더를 써요.

## 지원 프로바이더

LLM 클라이언트 추상화 계층이 프로바이더별 차이를 흡수하므로, 설정 값만 맞추면 AI 노드는 동일한 방식으로 호출해요.

## 프로바이더 추가하기

`/llm-configs` 에서 우측 상단의 `[+ Add Provider]` 를 눌러요. 프로바이더 유형에 따라 필수 필드가 조금씩 달라져요.

<Steps>
<Step>프로바이더 유형 드롭다운에서 OpenAI · Anthropic · Google AI · Azure OpenAI · Local 중 하나를 골라요.</Step>
<Step>별칭(이름)과 API Key를 입력해요. Azure · Local처럼 커스텀 엔드포인트가 필요한 경우 `Base URL` 을 함께 입력해요.</Step>
<Step>기본 모델은 `모델 불러오기` 버튼으로 프로바이더 API에서 받아온 목록에서 선택하거나, 원하는 모델 ID를 직접 입력해요 (예: `gpt-4o`, `claude-sonnet-4-6`, `llama3`). 생성 단계에서도 폼에 입력된 자격증명으로 미리 조회돼요.</Step>
<Step>`Test Connection` 을 눌러 `POST /api/llm-configs/:id/test` 로 경량 호출(모델 목록 조회 등)이 통과하는지 확인해요.</Step>
<Step>성공하면 저장해요. 가장 먼저 쓰는 프로바이더는 `⭐ 기본 프로바이더 설정` 으로 표시해 두면 AI 노드를 새로 만들 때 자동으로 선택돼요.</Step>
</Steps>

<Callout type="warn">API Key는 AES-256-GCM으로 암호화되어 저장돼요. 응답으로 돌아오는 값은 `sk-...xxxx` 처럼 마스킹되므로, 원본 키는 프로바이더 측 콘솔에서 따로 관리해 주세요.</Callout>

## 기본 모델 파라미터

프로바이더 단위로 기본 파라미터를 저장해 두면, AI 노드가 이 값을 초깃값으로 상속받아요. 노드의 설정 패널에서 개별적으로 덮어쓸 수 있어요.

```ts
{
  "provider": "local",
  "name": "Ollama (on prem)",

  },
  "isDefault": false
}
```
</Example>

## 팁 & 참고

- 기본 프로바이더는 `PATCH /api/llm-configs/:id/set-default` 한 번으로 바꿀 수 있어요. 이미 만든 AI 노드에는 영향을 주지 않고, 신규 노드에만 반영돼요.
- 임베딩을 지원하지 않는 프로바이더를 기본값으로 두어도 돼요. 지식 저장소는 컬렉션별로 임베딩 모델을 따로 지정하므로 충돌하지 않아요. 자세한 내용은 [지식 저장소](/docs/06-integrations-and-config/knowledge-base)를 참고해 주세요.
- AI 노드에서 파라미터를 바꾸는 법은 [AI 노드](/docs/02-nodes/ai)에서 설명해요.
- 저장된 설정의 모델 목록은 `GET /api/llm-configs/:id/models` 로 조회해요. 아직 저장 전(폼 단계)에는 `POST /api/llm-configs/preview-models` 에 `{ provider, apiKey, baseUrl? }` 를 보내 미리 조회할 수 있어요. preview 엔드포인트는 API Key를 저장하지 않고 요청 범위에서만 사용해요.
- Azure OpenAI는 모델 ID에 배포 이름(deployment name)을 입력해야 해요. 표준 모델 ID를 넣으면 404가 떨어져요.
