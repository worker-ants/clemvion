## `$input`: 직전 노드 결과

가장 자주 쓰는 참조예요. 현재 노드의 **입력 포트에 들어온 데이터**를 가리켜요.

<Example title="직전 노드의 email 필드 참조">
```ts
{{ $input.email }}
```
</Example>

<Example title="중첩 필드 접근">
```ts
{{ $input.user.profile.nickname }}
```
</Example>

<Callout type="warn">중간 경로에 `null`이 있으면 에러가 나요. 삼항 연산이나 `||`로 방어해요. 예: `{{ $input.user ? $input.user.name : "unknown" }}`</Callout>

## `$node["이름"]`: 임의 노드 결과 참조

`$input`은 **직전** 노드만 가리키지만, `$node`는 워크플로우 안의 **임의 노드** 결과를 끌어올 수 있어요.

<Example title="라벨로 참조">
```ts
{{ $node["HTTP Request"].output.data.items[0] }}
```
</Example>

<Example title="UUID로 참조 (라벨 변경에 안정적)">
```ts
{{ $node["3f2a9b1c-...-uuid"].output.result }}
```
</Example>

<Callout type="note">워크플로우 안의 노드 라벨은 고유해야 해요. 실수로 중복되면 실행 순서에 따라 `Label#2`, `Label#3` 같은 접미사가 자동으로 붙어요.</Callout>

## `$var`: 워크플로우 변수

Variable Declaration 노드로 선언한 변수는 `$var`로 읽고, Variable Modification 노드로 갱신해요.

<Example title="카운터 변수 참조">
```ts
{{ $var.counter + 1 }}
```
</Example>

## `$now`와 `$today`: 시간 참조

실행 시점 기준으로 평가되고, 같은 실행 내에서는 동일한 값이 유지돼요.

<Example title="현재 ISO 타임스탬프">
```ts
{{ $now }}
```
</Example>

<Example title="오늘 날짜로 파일 이름 만들기">
```ts
report-{{ $today }}.csv
```
</Example>

## `$env`: 환경 변수

셀프 호스팅 환경에서 워크스페이스 관리자가 허용한 키만 노출돼요. API 키, 외부 서비스 URL 등을 워크플로우에서 안전하게 참조할 때 써요.

<Example title="환경 변수 참조">
```ts
{{ $env.API_BASE_URL }}/users
```
</Example>

## `$trigger`: 트리거 페이로드

Webhook, Schedule, Manual 트리거가 제공하는 원본 데이터에 접근해요.

<Example title="웹훅 바디에서 이벤트 종류 읽기">
```ts
{{ $trigger.body.event }}
```
</Example>

## 팁 & 참고

- 자동완성은 마지막 실행 결과의 스키마를 기반으로 해요. 한 번도 실행하지 않은 워크플로우에서는 필드 제안이 제한되니 **먼저 한 번 실행**해 보는 걸 권장해요.
- 자동완성은 **현재 노드에서 접근 가능한 조상 노드와 해당 컨테이너 스코프 변수**만 보여줘요. 분기에서 반대쪽 가지의 노드나 다른 루프 내부 노드는 목록에서 제외돼요.
- 저장된 표현식이 접근 불가능한 노드를 참조하거나 컨테이너 밖에서 `$loop`·`$item`·`$itemIndex`를 쓰면 **주황색 경고**가 필드 하단에 표시돼요. 런타임에서 값이 비는 원인을 편집 시점에 미리 발견할 수 있어요.
- 실전 패턴은 [치트시트](/docs/03-expression-language/cheatsheet)에서 확인할 수 있어요.
- 실행 결과와 입력·출력 확인 방법은 [실행 결과](/docs/04-run-and-debug/run-results)에서 다뤄요.
