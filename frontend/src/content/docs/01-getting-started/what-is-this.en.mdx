This page explains what kind of tool this product is and what problems it solves.

## In one sentence

This product is a **visual builder that lets you create automation workflows without writing code**. Drop nodes onto the canvas one at a time, connect them with lines, and your repetitive work starts running on its own. Developers get advanced features like expressions, code editing, and custom nodes; non-developers get a smooth drag-and-drop experience.

## What can you build

- Automations that receive a request from an external service, then send an email or write to a database
- Schedule-based jobs that run on a fixed cadence to produce reports
- Flows where an AI classifies customer inquiries and routes them to the right team
- Simple apps that collect user input through a form, process it, and return a table or chart

Each block is a node. Nodes receive input, do their job based on configuration, and hand the result to the next node. The trigger is the **start signal** of the workflow, and the last node's output becomes the **final result**.

## Core building blocks

- **Sidebar**: jump between the big menus — Dashboard, Workflows, Triggers, Schedules, Integrations, Statistics, and more.
- **Workflow list**: see every automation you've built, toggle them on or off at a glance.
- **Canvas editor**: the space where you actually draw and configure a workflow. Pull nodes from the palette on the left, drop them on the canvas in the center, and fill in the details in the settings panel on the right.
- **Node system**: organized into seven categories — Trigger · Logic · Flow · AI · Integration · Data · Presentation.
- **Expression language**: pull results from earlier nodes with syntax like `{{ $input.field }}`.
- **Execution engine**: runs workflows safely and shows each node's state and result in real time.
