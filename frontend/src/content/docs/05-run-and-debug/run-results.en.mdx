## AI node Multi Turn results

AI Agent / Information Extractor runs in Multi Turn mode appear on the timeline as a collapsible tree. Expanding it shows **user messages, AI responses, and tool calls** inline as preview cards.

- While the conversation is active, a message-input UI sticks to the bottom of the inspector.
- After the conversation ends it reads like a chat thread. Clicking an item changes the detail panel's tabs:

## Browsing execution history

You can re-open results after a run has finished.

<Steps>
  <Step>In the editor header menu (`⋯`) pick `Execution history`, or click `View All Executions` in the Run Results drawer.</Step>
  <Step>On `/workflows/:id/executions`, narrow the list by status (All / Completed / Failed / Running / Cancelled / Waiting).</Step>
  <Step>Clicking a row jumps to `/workflows/:id/executions/:executionId`.</Step>
  <Step>From the detail page you can read the summary card, per-node results, errors, and navigate adjacent runs with `← Prev / Next →`.</Step>
</Steps>

You can also reach execution history from the Dashboard's Recent Executions widget and the Workflow List's context menu (`⋯ → Execution History`).

## Tips for inspecting per-node I/O

- **Copy buttons**: the Input/Output tabs expose `Copy Input` and `Copy Output` buttons. Paste the JSON straight into Run with Input to reproduce a run.
- **Re-run with this input**: from the execution detail, start a new run using a specific run's input.
- **Presentation node Preview**: Table, Chart, Carousel, and Template show a rendered preview rather than raw JSON. The JSON itself is still on the Output tab.

## Tips & notes

- To handle failures robustly, configure a per-node [error policy](/docs/05-run-and-debug/error-handling).
- Roll back to an earlier state via [Version history](/docs/05-run-and-debug/version-history).
- For expression-related errors, check [Expression basics](/docs/04-expression-language/basics) and [Variables and context](/docs/04-expression-language/variables-and-context) for likely causes.
