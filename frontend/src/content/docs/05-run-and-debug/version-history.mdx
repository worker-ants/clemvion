이 페이지에서는 에디터 안에서 **워크플로우 버전을 만들고 비교하고 복원**하는 방법을 설명해요. 과거 상태로 안전하게 되돌아가고 싶을 때 써요.

## 버전은 언제 만들어지나요?

**별도 버튼 조작 없이** 캔버스를 저장(`Save`)할 때마다 서버가 자동으로 스냅샷을 기록해요.

- 저장 트랜잭션 직후 같은 시점의 노드·연결선 전체 상태가 `jsonb` 스냅샷으로 보관돼요.
- 저장된 버전은 **불변**이에요. 이후 캔버스를 수정해도 과거 버전은 변하지 않아요.
- 저장 요청에 `changeSummary` 메모를 넣으면 버전 이력에 그대로 표시돼요.

<Callout type="tip">중요한 변경 전에 `Save`를 한 번 누르는 습관을 들이면, 롤백 지점을 스스로 찍어둘 수 있어요.</Callout>

## Version History 패널 열기

<Steps>
  <Step>에디터 우측 상단 툴바의 `⋯` (More) 메뉴를 열어요.</Step>
  <Step>`Version History`를 클릭하면 캔버스 오른쪽에 사이드 패널이 열려요.</Step>
  <Step>패널에는 최신 버전이 위에 오도록 `v3, v2, v1` 순서로 나열돼요.</Step>
  <Step>각 항목에는 버전 번호, 작성자, 생성 시각, 변경 요약이 표시되고 `상세(👁)`·`복원(↺)` 액션 버튼이 있어요.</Step>
</Steps>

버전이 하나도 없을 때는 "No versions yet. Save the canvas to create the first version." 안내가 나와요. 한 번 `Save`를 누르면 첫 버전이 만들어져요.

## 버전 상세 보기

버전 항목의 `👁 (Eye)` 버튼을 누르면 읽기 전용 상세 다이얼로그가 열려요.

- 워크플로우 메타: 이름, 설명
- 노드 목록: 라벨, 타입, 좌표, 비활성 여부
- 연결선 목록: `source:port → target:port` 쌍

복원 전에 "이 버전 맞나?"를 안전하게 확인할 때 써요.

## 두 버전 비교 (Diff)

Compare 토글을 켜면 각 버전 항목에 체크박스가 나타나요.

<Steps>
  <Step>패널 상단의 `☐ Compare versions` 토글을 활성화해요.</Step>
  <Step>비교할 두 버전을 체크박스로 선택해요. 낮은 버전이 `before`, 높은 버전이 `after`가 돼요.</Step>
  <Step>`Diff` 버튼을 클릭하면 비교 다이얼로그가 열려요.</Step>
</Steps>

Diff 다이얼로그는 다음 항목을 보여줘요.

## 과거 버전으로 복원

복원은 과거 스냅샷으로 **현재 캔버스를 덮어쓰는** 작업이에요.

<Steps>
  <Step>되돌리고 싶은 버전 항목에서 `↺` (Restore) 버튼을 클릭해요.</Step>
  <Step>확인 다이얼로그의 안내를 읽고 `확인`을 눌러요.</Step>
  <Step>복원이 완료되면 페이지가 리로드되고, 캔버스가 선택한 버전의 상태로 바뀌어요.</Step>
</Steps>

<Callout type="note">복원 동작은 **새로운 버전**으로 자동 기록돼요. 변경 요약은 항상 `Restored from vN` 형식이에요. 그래서 복원 이후에도 다시 이전 상태로 되돌아갈 수 있어요.</Callout>

<Callout type="warn">복원은 편집 중인 내용을 덮어써요. 아직 저장하지 않은 변경이 있다면 먼저 `Save`로 현재 상태를 버전으로 남겨두는 게 안전해요.</Callout>

## 팁 & 참고

- 워크플로우가 삭제되면 모든 버전도 함께 삭제돼요 (`ON DELETE CASCADE`).
- 복원된 버전의 `change_summary`는 항상 `Restored from vN`이라 이력에서 바로 식별할 수 있어요.
- 실행 이력은 버전과는 별개로 관리돼요. 각 실행 결과는 [실행 결과](/docs/05-run-and-debug/run-results)에서 확인해요.
- 실행 중단과 재실행은 [워크플로우 실행하기](/docs/05-run-and-debug/running-a-workflow)를 참고해요.
