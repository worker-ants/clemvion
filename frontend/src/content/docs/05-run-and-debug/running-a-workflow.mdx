이 페이지에서는 에디터에서 워크플로우를 **실행하는 세 가지 방식**과, 트리거 종류(Manual / Schedule / Webhook)에 따른 차이를 설명해요.

## 세 가지 실행 방식

에디터 상단 툴바의 `▶ Run` 버튼 드롭다운에서 선택할 수 있어요.

<FieldTable rows={[
  { name: "Run", type: "전체 실행", description: "빈 입력(또는 Mock Input 설정값)으로 워크플로우 루트 노드부터 끝까지 실행해요. 단축키는 Ctrl+Enter예요." },
  { name: "Run with Input", type: "입력과 함께 실행", description: "테스트 입력 JSON을 직접 편집해 넣고 실행해요. 이전 실행의 입력을 불러오거나, 테스트 세트로 저장해 재사용할 수도 있어요." },
  { name: "Run from Selected", type: "부분 실행", description: "선택한 노드부터 하류 끝까지만 실행해요. 이전 실행에서 해당 노드가 받았던 입력을 그대로 쓰거나 수동 입력을 제공할 수 있어요." }
]} />

추가로 노드를 우클릭하면 나오는 **단일 노드 실행**(이 노드만 실행)도 쓸 수 있어요. 노드의 동작을 빠르게 검증할 때 유용해요.

## Run with Input 다이얼로그

`Run with Input`을 선택하면 테스트 입력 다이얼로그가 열려요.

<Steps>
  <Step>JSON 에디터에 테스트 데이터를 직접 입력해요. 실시간으로 유효성 검증이 이루어져요.</Step>
  <Step>`Load from History` 드롭다운으로 이전 실행의 입력 데이터를 불러올 수 있어요.</Step>
  <Step>자주 쓰는 입력은 이름을 붙여 저장해두면 다음부터 원클릭으로 재사용할 수 있어요.</Step>
  <Step>`Run ▶`을 눌러 실행을 시작해요.</Step>
</Steps>

<Example title="Run with Input JSON 예시">
```json
{
  "userId": 123,
  "action": "signup",
  "data": {
    "email": "test@example.com"
  }
}
```
</Example>

## Run from Selected

디버깅 중인 노드 하나만 집중해서 확인하고 싶을 때 써요.

<Steps>
  <Step>캔버스에서 출발점이 될 노드를 선택해요.</Step>
  <Step>노드 우클릭 → `여기서부터 실행`을 선택해요.</Step>
  <Step>이전 실행에서 해당 노드가 받았던 입력이 자동으로 채워져요. 필요하면 수동으로 수정할 수 있어요.</Step>
  <Step>선택 노드부터 연결된 하류 노드까지만 실행돼요.</Step>
</Steps>

<Callout type="tip">AI Agent나 HTTP Request처럼 비용·시간이 드는 노드를 다시 돌리고 싶지 않을 때, 그 **다음 노드**부터 부분 실행하면 시간을 크게 아낄 수 있어요.</Callout>

## 트리거 종류와 실행 방식의 차이

**트리거**는 워크플로우가 실행되는 **시작 조건**이에요. 에디터의 Run 버튼은 어떤 트리거든 관계없이 수동으로 실행을 걸지만, 배포 이후에는 트리거별로 자동 실행돼요.

<FieldTable rows={[
  { name: "Manual Trigger", type: "수동", description: "사용자가 Run 버튼을 누를 때만 실행돼요. 테스트·디버깅·일회성 작업에 적합해요. `$execution.mode` 값은 `manual`이에요." },
  { name: "Schedule", type: "스케줄", description: "Cron 표현식 기반으로 주기적으로 실행돼요. 배치 작업, 정기 리포트, 데이터 수집에 써요. `$execution.mode` 값은 `schedule`이에요." },
  { name: "Webhook", type: "웹훅", description: "외부 시스템이 HTTPS로 호출하면 실행돼요. 페이로드는 `$trigger.body`로 들어와요. 이벤트 기반 자동화에 써요. `$execution.mode` 값은 `webhook`이에요." }
]} />

<Callout type="note">에디터에서 `Run`을 누르면 트리거 종류와 상관없이 항상 `manual` 모드로 실행돼요. Webhook 페이로드 시뮬레이션이 필요하다면 `Run with Input`에서 JSON을 직접 넣어 주세요.</Callout>

## 실행 상태 확인

실행을 시작하면 에디터 상단에 실행 상태 바가 표시돼요.

```
▶ Running...   0.5s elapsed    [■ Stop]
```

- 현재 실행 중인 노드: 파란 펄스 테두리
- 완료된 노드: 초록 체크 + 소요 시간
- 엣지: 데이터 흐름 애니메이션
- 분기 노드: 실제 실행된 경로만 진하게, 미실행 경로는 흐릿하게

하단의 **Run Results 드로어**가 자동으로 펼쳐지며, 각 노드의 입력·출력·상태가 타임라인으로 나타나요. 자세한 내용은 [실행 결과](/docs/05-run-and-debug/run-results)에서 다뤄요.
