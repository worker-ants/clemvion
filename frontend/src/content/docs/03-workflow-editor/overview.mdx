이 페이지에서는 캔버스 안에 내장된 **AI 어시스턴트**가 무엇이고, 어떻게 여닫는지, 어떤 흐름으로 워크플로우를 만들거나 고쳐 주는지를 정리해요. 실제로 자연어로 요청을 던져 4-노드 자동화를 완성하는 시연은 [Walkthrough](/docs/03-workflow-editor/walkthrough)에서 이어져요.

## 한 문장으로 요약

AI 어시스턴트는 **자연어 요청을 받아 노드와 연결선을 직접 만들어 주는 채팅형 도우미**예요. "주문 취소 프로세스 추가해줘" 같은 모호한 요청도 명확화 → 계획 제안 → 실행 3단계로 풀어 가며, 각 단계에서 사용자가 개입할 수 있게 멈춰 세워요.

## 어디에 있고 어떻게 여나요?

- **위치**: 워크플로우 에디터 우측에 도킹된 360px 패널이에요. 노드를 클릭하면 보이는 설정 패널과 같은 자리를 쓰므로, 어시스턴트를 열면 노드 선택이 풀리고 어시스턴트가 닫히면 다시 설정 패널을 열 수 있어요.
- **단축키**: `Cmd/Ctrl + /` 가 어시스턴트 패널을 켜고 끄는 토글이에요. 캔버스 어디서든 동작해요.
- **기본 상태**: 닫힘. 패널을 처음 열면 빈 화면 안내가 나와요.

<Callout type="note">툴바에 별도 버튼은 두지 않았어요. 단축키 한 번이면 어디서든 어시스턴트를 부를 수 있어요.</Callout>

### 빈 화면 안내

처음 패널을 열면 다음 안내가 보여요.

- 제목: "무엇을 도와드릴까요?"
- 서브: "자연어로 원하는 워크플로우를 설명하면 함께 만들어드려요."
- 예시 버튼 4개 — 한 번 누르면 자동으로 메시지로 채워져요.
  - "주문 취소 프로세스 추가해줘"
  - "HTTP 노드에 Authorization 헤더 추가"
  - "현재 워크플로우를 검토하고 개선점 제안해줘"
  - 그 외 1건 (배치 정리 예시)

### Plan 카드 UX

② 단계에서 계획 카드는 다음 요소로 구성돼요.

- **제목**과 **요약** — 무엇을 만들지 한 줄로 정리해요.
- **단계 체크박스 리스트** — 각 단계는 처음에 ◯(대기)로 표시되고, 실행되면 ✓(완료, 초록)로, 실패하면 ⚠(빨강)로 바뀌어요.
- **답변이 필요한 항목** — 어시스턴트가 추가 결정을 사용자에게 맡길 때 호박색 안내 박스로 질문을 보여줘요. 답은 **하단 메시지 입력창**에 평소처럼 적어 보내면 돼요. 별도 답변 폼은 없어요.
- **"계획대로 진행" 버튼** — 사용자가 누르면 ③ 단계가 시작돼요. 누르기 전엔 캔버스가 변하지 않아요.

<Callout type="tip">
계획에 동의하기 어려운 부분이 있다면 승인 버튼 대신 채팅으로 "X를 빼고 Y를 더 넣어 줘" 처럼 보내요. 어시스턴트가 새 계획 카드로 교체해서 다시 제시해요.
</Callout>

## 세션과 영속성

어시스턴트의 대화는 **워크플로우 단위로 한 세션**이 만들어져요. 세션은 서버에 저장되므로 페이지를 새로고침하거나 다른 기기에서 접속해도 이어서 쓸 수 있어요.

- **자동 생성** — 첫 메시지를 보내면 세션이 자동으로 만들어져요. 빈 패널에서 `+` 버튼으로 새 세션을 만들 수도 있어요.
- **자동 제목** — 첫 사용자 메시지의 앞부분으로 세션 제목이 자동으로 생성돼서 패널 헤더에 작은 글씨로 보여요.
- **이력 보관** — 한 워크플로우당 최근 세션 50개까지 보관돼요. v1에서는 패널 안에 세션 전환 UI가 노출되지 않으므로, 같은 워크플로우의 이전 세션을 다시 열려면 `+`로 새 세션을 만들거나 자동 복원되는 가장 최근 세션을 그대로 이어가요.
- **다른 사용자와 공유 안 됨** — 세션은 개인 단위예요. 팀 워크스페이스의 다른 멤버는 자기 세션을 따로 가져요.

## 모델 선택

패널 상단의 **모델** 드롭다운으로 세션 도중에도 LLM 프로바이더를 바꿀 수 있어요. 기본값은 워크스페이스에 등록한 LLM 설정 중 우선순위가 높은 항목이에요. 모델을 바꾸면 다음 메시지부터 적용돼요. LLM 설정 등록은 [LLM 설정](/docs/06-integrations-and-config/llm-config)에서 다뤄요.

## 실행 결과로 진단·수정하기

어시스턴트는 직접 실행하지 못하지만, **사용자가 `Run` 으로 실행한 결과**를 읽고 어디서 무엇이 잘못됐는지 짚어 줄 수 있어요.

<Steps>
<Step>실행이 끝난 뒤 어시스턴트에 "왜 실패했어?" 또는 "Lead Router 노드 결과 확인해 줘" 처럼 물어요.</Step>
<Step>어시스턴트가 최근 실행 목록과 상세 타임라인을 조회해 실패한 노드와 직전 노드의 입출력을 확인해요. (민감 필드는 자동으로 마스킹돼요.)</Step>
<Step>원인이 짚이면 수정 계획 카드를 띄워요. 사용자가 승인하면 노드 설정을 패치해요.</Step>
<Step>사용자가 다시 `Run` 버튼을 눌러 결과를 확인해요.</Step>
</Steps>

## 자주 만나는 오류

| 상황 | 화면에 보이는 메시지 | 사용자 조치 |
| --- | --- | --- |
| LLM 설정이 등록되지 않았을 때 | "LLM 설정을 먼저 등록해 주세요." 안내와 LLM 설정 화면 링크 | 링크를 눌러 [LLM 설정](/docs/06-integrations-and-config/llm-config) 에서 OpenAI 등 프로바이더를 등록 |
| 선택한 LLM 설정이 삭제됐을 때 | "선택한 LLM 설정을 찾을 수 없어요. 다시 선택해 주세요." | 패널 상단 모델 드롭다운에서 다른 설정을 고르거나 워크스페이스 기본값을 사용 |
| 프로바이더 rate limit | 빨간 오류 박스 — "잠시 후 다시 시도해 주세요." | 잠시 기다린 뒤 재전송하거나 "이어서 진행" 버튼으로 같은 흐름을 재개 |
| 응답 지연 / 타임아웃 | "응답이 늦어지고 있어요. 다시 시도할까요?" | 재시도 버튼을 누르거나 새 메시지 전송 |
| 도구 호출 실패 | 메시지 아래 빨간 배지에 코드와 짧은 설명 | 어시스턴트가 다음 턴에 자동으로 다시 시도해요. 반복 실패 시 사용자가 직접 노드를 수정하거나 다른 접근을 지시해요. |
| 한 턴의 도구 호출 한도 초과 | "이어서 진행" 버튼이 입력창 위에 노출 | 버튼을 누르면 같은 흐름이 다음 턴으로 이어져요 |
| 세션 만료 / 외부에서 삭제 | "세션이 만료되었어요. 새 대화를 시작할게요." | 새 세션이 자동으로 만들어지므로 이어서 메시지를 보내면 돼요 |

## 팁 & 참고

- 실제 사용 흐름은 [Walkthrough](/docs/03-workflow-editor/walkthrough) 에서 자연어 한 줄로 4-노드 워크플로우를 완성하는 과정을 단계별로 따라가요.
- AI 어시스턴트와 [AI Agent 노드](/docs/02-nodes/ai)는 별개예요. **어시스턴트는 에디터 안의 도우미**이고, **AI Agent 노드는 워크플로우 안에서 실행되는 LLM 호출 단위**예요. 어시스턴트가 만들어 주는 결과물 안에 AI Agent 노드가 포함될 수도 있어요.
- 어시스턴트가 만든 변경은 일반 편집과 같은 Undo 스택에 들어가요. 마음에 들지 않으면 `Ctrl+Z` 로 되돌리고 다시 지시하세요.
- 도구가 무엇을 했는지 항상 메시지 아래 배지로 표시되니, 어시스턴트의 행동이 의심스러우면 배지를 확인해요.
