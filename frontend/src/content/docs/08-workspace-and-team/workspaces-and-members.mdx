워크스페이스는 워크플로우 · Integration · 지식 베이스가 묶이는 단위예요. 혼자 쓰는 **개인 워크스페이스**와 여러 명이 함께 쓰는 **팀 워크스페이스** 두 가지가 있고, 사이드바 하단의 사용자 메뉴에서 언제든 전환할 수 있어요.

## 개인 vs 팀

| 구분 | 개인 워크스페이스 | 팀 워크스페이스 |
| ---- | ----------------- | --------------- |
| 멤버 수 | 1명 (본인) | 여러 명 |
| 자동 생성 | 첫 가입·이메일 인증 시 1개가 자동 생성 | 사용자가 직접 만든다 |
| 멤버 관리 | 없음 | Admin+ 가 초대/역할 변경/제거 |
| 공유 워크플로우 표시 | 없음 (모두 내 것) | 모든 워크플로우에 `Team` 배지 |
| Owner 이양·삭제 | 없음 (탈퇴=계정 삭제) | Owner 가 이양 또는 삭제 가능 |

> 초대 토큰으로 가입한 사용자는 **개인 워크스페이스가 자동 생성되지 않아요**. 초대된 팀 워크스페이스가 첫 컨텍스트가 되고, 필요하면 워크스페이스 관리 화면에서 직접 만들 수 있어요.

## 팀 워크스페이스 만들기

1. 사이드바 사용자 아바타 클릭 → 워크스페이스 섹션의 `+ 새 워크스페이스`.
2. 이름을 정하고 만들면 자동으로 본인이 **Owner** 가 돼요.
3. 워크스페이스가 전환되고, 좌측 사이드바 메뉴가 새 컨텍스트에 맞춰 갱신돼요.

## 역할 (RBAC)

| 역할 | 권한 요약 |
| ---- | --------- |
| **Owner** | 모든 권한 + 워크스페이스 삭제·소유권 이양 |
| **Admin** | 멤버 초대/제거, 설정 변경, 모든 리소스 CRUD |
| **Editor** | 워크플로우·트리거·스케줄 CRUD + 실행 |
| **Viewer** | 모든 리소스 읽기 전용 |

권한 상세는 `spec/2-navigation/9-user-profile.md` §4.2 참고.

## 멤버 초대하기

**워크스페이스 관리 → 멤버** 탭에서 `[email@…] [Role ▼] [초대]` 폼을 사용해요. 두 가지 경로가 있어요:

- **이미 가입한 사용자** — 이메일이 일치하면 즉시 멤버로 추가되고, 알림이 전송돼요.
- **미가입자** — 7일 유효한 초대 토큰이 메일로 발송돼요. 수신자가 가입을 마치는 순간 자동으로 워크스페이스에 합류해요.

대기 중 초대 목록에는 만료 시각이 함께 표시되고, 각 행에서 다음을 할 수 있어요:

- `재발송` — 기존 토큰을 즉시 무효화하고 새 토큰으로 메일을 다시 보내요. 만료 시계도 발송 시점부터 다시 7일.
- `초대 취소` — 대기 중 초대를 폐기해요. 이미 수락된 초대는 취소할 수 없어요.

> 같은 이메일을 다시 초대하면 기존 토큰이 자동으로 무효화되고 새 토큰이 발급돼요 — 다중 토큰이 동시에 살아있지 않도록 단일 초대 row 를 유지해요.

## 초대를 받은 사용자

메일의 `초대 수락하기` 버튼은 회원가입 페이지를 `?invitationToken=…` 쿼리와 함께 열어줘요.

- **미가입자**: 이메일이 prefill 되고 잠겨 있어요 (다른 이메일로는 가입할 수 없어요 — 이는 토큰 누출 시 권한 escalate 를 차단하기 위한 의도적 제약). 비밀번호·이름만 입력하면 가입과 동시에 워크스페이스에 자동 합류, 곧바로 대시보드로 이동해요. **별도 이메일 인증 단계는 없어요** — 토큰 자체가 이메일 소유 증명이거든요.
- **이미 가입한 사용자**: 본인 이메일과 토큰 이메일이 같으면 `[수락]` 버튼을 통해 합류해요. 다른 이메일로 로그인한 상태라면 안내 메시지가 나타나고 해당 계정으로 다시 로그인해야 해요.
- **만료/이미 사용된 토큰**: 가입 페이지에 안내 배너가 표시돼요. 워크스페이스 Admin 에게 재발송을 요청해 주세요.

## 공유 워크플로우 (Team 배지)

팀 워크스페이스로 전환하면 워크플로우 목록의 모든 항목에 `👥 Team` 배지가 붙어요. 이는 "이 워크플로우는 같은 팀의 다른 멤버가 보고 편집할 수 있다" 라는 시각적 표시예요. 개인 워크스페이스에서는 배지가 보이지 않아요.

### 소유 필터

팀 워크스페이스에서는 검색 우측에 `전체 / 내 워크플로우 / 공유된 워크플로우` 세 가지 필터 그룹이 추가로 나타나요. 같은 팀 안에서 "내가 만든 것" 과 "남이 만든 것" 을 분리해 보는 데 써요. 개인 워크스페이스에서는 이 그룹이 비노출돼요.

## 워크스페이스 떠나기·삭제

- **나가기** (`위험 영역` 탭) — 본인 멤버십만 제거해요. 유일한 Owner 라면 차단되니, 먼저 [Owner 이양] 으로 다른 멤버에게 권한을 넘기거나 [워크스페이스 삭제] 를 사용해야 해요.
- **Owner 이양** — 같은 워크스페이스의 비-Owner 멤버 한 명을 골라 권한을 넘겨요. 트랜잭션 내에서 두 멤버의 역할이 동시에 swap (대상 → Owner, 기존 → Admin) 돼요.
- **삭제** — Owner 만 가능. 이름을 재입력해 확인하면 멤버·초대·워크스페이스가 트랜잭션으로 정리돼요.
