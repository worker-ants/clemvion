이 페이지에서는 워크플로우의 결과를 **시각화**하거나 **사용자 입력**을 받는 프리젠테이션 노드 다섯 가지를 설명해요. Carousel·Table·Chart·Template은 버튼을 달아 Human-in-the-loop로 동작시킬 수도 있고, Form은 기본이 블로킹 입력 노드예요.

## 공통 패턴 — 버튼과 블로킹 모드

Carousel, Table, Chart, Template 네 노드는 공통 `buttons` 필드를 갖고 있어요. 버튼이 하나라도 설정되면 실행이 **waiting_for_input** 상태로 진입해서 사용자의 클릭을 무제한 대기해요.

### 언제 써요

- 데이터를 **슬라이드** 형태로 보여주고 싶을 때 써요.
- **Static** 모드에서 슬라이드를 직접 정의하거나, **Dynamic** 모드에서 배열 데이터를 자동 매핑할 수 있어요.

### 예시

<Example title="Dynamic 모드 + 아이템 버튼">
```ts

  mode: "dynamic",
  source: "{{ $node[\"API\"].output.results }}",
  titleField: "name",
  descriptionField: "summary",
  imageField: "thumbnail",

    { label: "선택", type: "port", style: "primary" }

  clickedAt: "…",
  selectedItem: { title: "…", description: "…" },
```

## Table

### 언제 써요

- 데이터를 **테이블** 형태로 보여주고, 정렬·페이지네이션을 적용하고 싶을 때 써요.
- Static 모드로 표를 직접 구성하거나, Dynamic 모드로 배열 데이터 소스에서 매핑할 수 있어요.

### 필드

  {name: "rows", required: false, type: "RowDef[]", description: "Static 모드 행 데이터예요. 각 행은 컬럼 field를 키로 하는 문자열 객체예요.", default: "[]"},

### 예시

<Example title="Dynamic 모드 + per-item 표현식 컬럼">
```ts
// config
{
  mode: "dynamic",
  dataSource: "{{ $node[\"API\"].output.users }}",
  columns: [
    { field: "name", label: "이름", width: "200px" },
    { field: "{{ $sourceItem.first + ' ' + $sourceItem.last }}", label: "전체 이름" },
    { field: "score", label: "점수", sortable: true, format: "0,0.00" }
  ],
  pagination: true,
  pageSize: 20,
  sortBy: "score",
  sortOrder: "desc"
}
```
</Example>

<Callout type="note">
Table은 기본적으로 페이지네이션이 켜져 있어요. 대량 데이터를 표시할 때 최대 200행/페이지를 강제하니, 필요한 양만 안전하게 보여줘요.
</Callout>

```ts

  title: "Approval Request",
  description: "아래 내용을 확인하고 **승인 여부**를 선택해요.",
  submitLabel: "Submit",
```

<Callout type="warn">
Form 실행은 **폼 제출 시까지 무제한 대기**해요. 실행 상태가 `waiting_for_input`으로 바뀌며, 외부 cancel/종료 외에는 타임아웃이 발생하지 않아요. 실행 파일(.exe, .sh), 스크립트(.js, .py), 아카이브(.zip) 등은 기본 허용 MIME 목록에 포함되지 않으니 필요 시 명시적으로 확장해요.
</Callout>


</Callout>

## 캔버스 요약

- Carousel: `{layout} · {titleField}` 또는 `{layout} · {N} buttons`
- Table: `{N} columns · pagination` 또는 `{N} columns · {N} buttons`
- Chart: `{chartType} · {xAxis.field} / {yAxis.field}` 또는 `{chartType} · {N} buttons`
- Form: `{N} fields · "{title}"`
- Template: `{outputFormat} · {N} lines` 또는 `{outputFormat} · {N} buttons`

## 팁 & 참고

- 실행 결과 드로어에서 각 프리젠테이션 노드가 어떻게 렌더링되는지는 [실행 결과](/docs/04-run-and-debug/run-results)에서 다뤄요.
- 버튼 블로킹·폼 대기·AI 대화가 섞인 워크플로우에서 실행 상태가 `waiting_for_input`으로 바뀌면, 사용자가 버튼을 누르거나 폼을 제출하기 전까지 실행이 일시정지돼요. 사용자의 응답이 들어오면 자동으로 이어서 실행돼요.
- 파워유저 팁: 버튼이 있는 Presentation 노드는 버튼마다 전용 출력 포트가 자동으로 생겨요. 사용자가 누른 버튼의 포트로만 흐름이 이어지므로, 버튼별 후속 동작을 분기해서 디자인할 수 있어요.
