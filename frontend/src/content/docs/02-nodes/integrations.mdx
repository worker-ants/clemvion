이 페이지에서는 외부 서비스와 연동하는 통합 노드 세 종류를 설명해요. 대부분의 통합 노드는 공통적으로 **Integration 엔티티**를 참조해 인증과 연결 정보를 재사용하고, 성공/실패를 각각 다른 출력 포트로 분기해요.

## 공통 패턴

### Integration 참조

각 노드에 공통으로 있는 `integrationId` 필드는 Integration 엔티티(사이드바 → Integrations)에 저장된 인증 정보를 참조해요. 자격 증명은 AES-256-GCM으로 자동 암·복호화돼요. 연결 상태가 `connected`가 아니면 해당 실행은 `INTEGRATION_NOT_CONNECTED`로 실패해요.

## HTTP Request

### 언제 써요

- 임의의 **HTTP API를 호출**하고 싶을 때 써요.
- Integration을 참조하면 인증 헤더가 자동으로 주입돼요. 인증 없이도 호출할 수 있어요.

### 필드

<FieldTable rows={[
  {name: "method", required: true, type: "enum", description: "GET / POST / PUT / PATCH / DELETE / HEAD / OPTIONS 중 하나예요.", default: "GET"},
  {name: "url", required: true, type: "expression", description: "요청 URL이에요. Integration에 `base_url`이 있으면 상대 경로를 써서 자동으로 합칠 수 있어요.", default: "-"},
  {name: "authentication", required: true, type: "`none` / `integration` / `custom`", description: "인증 방식이에요.", default: "none"},
  {name: "integrationId", required: false, type: "UUID", description: "`authentication=integration`일 때 필수예요.", default: "-"},
  {name: "headers", required: false, type: "KeyValue[]", description: "요청 헤더예요.", default: "[]"},

  {name: "body", required: false, type: "object", description: "요청 본문이에요.", default: "-"},
  {name: "bodyType", required: false, type: "enum", description: "`json` / `form-data` / `x-www-form-urlencoded` / `raw` / `binary` 중 하나예요.", default: "json"},

  {name: "timeout", required: false, type: "integer(ms)", description: "요청 타임아웃이에요.", default: "30000"},
  {name: "followRedirects", required: false, type: "boolean", description: "리다이렉트 따라가기 여부예요.", default: "true"},
  {name: "verifySsl", required: false, type: "boolean", description: "SSL 인증서 검증 여부예요.", default: "true"}
]} />

### 포트

- `success` — HTTP 2xx 응답일 때
- `error` — 4xx/5xx 또는 네트워크 오류일 때

### 예시

<Example title="외부 API 호출 + Bearer 토큰 인증">
```ts
// config
{
  method: "POST",
  url: "/tickets",                 // base_url이 credential에 있으면 자동 결합
  authentication: "integration",
  integrationId: "…",
  headers:    [{ key: "Accept", value: "application/json" }],
  bodyType:   "json",
  body:       { title: "{{ $input.subject }}", body: "{{ $input.message }}" },
  timeout:    30000
}

// success 포트
{ data: { id: "TCK-1" }, meta: { statusCode: 201, headers: {...}, duration_ms: 250 } }

// error 포트
{ error: { statusCode: 500, message: "Internal Server Error", body: {...} }, meta: { duration_ms: 100 } }
```
</Example>

<Callout type="tip">
Integration의 `auth_type`에 따라 헤더/쿼리가 자동 주입돼요. api_key(header/query), bearer_token, basic을 지원해요. 노드에서 직접 입력한 `headers`는 credential 주입 헤더를 덮어써요(뒤가 우선).
</Callout>

## Database Query

### 언제 써요

- PostgreSQL 또는 MySQL에 **SQL을 직접 실행**하고 싶을 때 써요.
- SELECT로 데이터를 읽거나, INSERT/UPDATE/DELETE로 조작할 수 있어요.

### 필드

<FieldTable rows={[
  {name: "integrationId", required: true, type: "UUID", description: "DB Integration 참조예요. `driver`는 `postgres` 또는 `mysql`이어야 해요.", default: "-"},
  {name: "queryType", required: true, type: "enum", description: "`select` / `insert` / `update` / `delete` / `raw` 중 하나예요.", default: "select"},
  {name: "query", required: true, type: "expression", description: "실행할 SQL이에요. 바인딩은 `$1`, `$2`, …를 써요. MySQL 드라이버는 내부적으로 `?`로 변환해요.", default: "-"},
  {name: "parameters", required: false, type: "array 또는 JSON 문자열", description: "플레이스홀더에 순서대로 바인딩할 값이에요. JSON 문자열을 넣으면 자동으로 배열로 파싱돼요.", default: "[]"}
]} />

### 포트

- `success` — 쿼리 정상 완료
- `error` — 런타임 실패(연결/쿼리 오류). 사전 설정 오류는 포트로 가지 않고 워크플로우 전체 실패로 처리돼요.

### 예시

<Example title="SELECT + 파라미터 바인딩">
```ts
// config
{
  integrationId: "…",
  queryType: "select",
  query: "SELECT id, name FROM users WHERE tenant_id = $1 AND status = $2",
  parameters: ["{{ $input.tenantId }}", "active"]
}

// success 출력
{
  rows: [{ id: 1, name: "Kim" }, ...],
  rowCount: 42,
  fields: [{ name: "id", dataTypeID: 23 }, ...],
  query: "…",
  queryType: "select",
  durationMs: 18,
  status: "ok"
}
```
</Example>

<Callout type="warn">
사전 설정 오류(`INTEGRATION_INCOMPLETE`, `INVALID_PARAMETERS`, `INTEGRATION_SERVICE_UNAVAILABLE`)는 **throw되어 실행 전체가 실패**해요. 반면 SQL 실행 중 발생한 오류는 `error` 포트로 라우팅돼요. 분기 처리하고 싶은 오류는 `error` 포트에 이어서 노드를 연결해요.
</Callout>

## Send Email

### 언제 써요

- **SMTP**를 통해 이메일을 발송하고 싶을 때 써요.
- To/CC/BCC, 제목, 본문(텍스트·HTML), 첨부를 지원해요.

### 필드

<FieldTable rows={[
  {name: "integrationId", required: true, type: "UUID", description: "SMTP Integration 참조예요. host, port, secure, username, password, default_from이 필수예요.", default: "-"},
  {name: "to", required: true, type: "string[]", description: "수신자 이메일 목록이에요. 각 항목은 표현식을 허용해요.", default: "[]"},
  {name: "cc", required: false, type: "string[]", description: "참조 수신자예요.", default: "[]"},
  {name: "bcc", required: false, type: "string[]", description: "숨은 참조 수신자예요.", default: "[]"},
  {name: "subject", required: true, type: "expression", description: "메일 제목이에요.", default: "-"},
  {name: "body", required: true, type: "expression", description: "메일 본문이에요.", default: "-"},
  {name: "bodyType", required: false, type: "`text` / `html`", description: "본문 형식이에요.", default: "text"},
  {name: "attachments", required: false, type: "Attachment[]", description: "첨부 목록이에요. 각 항목은 `filename`, `content`(Base64 또는 URL)를 가져요.", default: "[]"}
]} />

### 포트

- `out` — 발송 결과

### 예시

<Example title="환영 메일 발송">
```ts
// config
{
  integrationId: "smtp-integration-id",
  to: ["{{ $input.user.email }}"],
  subject: "{{ $input.user.name }}님, 가입을 환영해요",
  body: "<h1>안녕하세요</h1><p>가입해 주셔서 고마워요.</p>",
  bodyType: "html"
}

// 출력
{
  data: {
    messageId: "<…@smtp.example.com>",
    accepted: ["user@example.com"],
    rejected: [],
    to: ["user@example.com"],
    cc: [],
    subject: "…",
    bodyType: "html",
    status: "sent",
    durationMs: 412
  }
}
```
</Example>

<Callout type="note">
To/CC/BCC는 배열과 쉼표 구분 문자열 모두 허용해요. 내부적으로 trim 후 빈 값은 제거하고, 최종적으로 수신자가 하나도 없으면 `"No valid recipients"` 오류로 실패해요.
</Callout>

## 캔버스 요약

- HTTP Request: `{method} {url}` (URL 35자 초과 시 잘림)
- Database Query: `{queryType} · {쿼리 첫 줄}` (잘림)
- Send Email: `to: {수신자}` (2명 초과 시 `+N`)

Integration이 삭제되면 캔버스 요약에 `⚠ Missing integration` 배지가 붙어요.

## 팁 & 참고

- Integration 추가와 상태 관리는 [통합 관리](/docs/05-integrations-and-config/integration-management)에서 다뤄요.
- 실행 시 각 호출 결과는 Usage 로그에 기록돼요. `status`, `durationMs`, `error.code`를 통해 실패 원인을 추적할 수 있어요.
- 개발자용 팁: HTTP Request의 헤더 우선순위는 `credentials.default_headers` → credential 주입 헤더 → 노드 `headers` 순이에요(뒤가 우선). base_url이 있을 때 노드의 `url`이 절대 URL이 아니면 자동으로 결합돼요.
