이 페이지에서는 **Workflow 노드**로 다른 워크플로우를 서브 워크플로우처럼 호출하는 방법을 설명해요. 재사용 가능한 하위 흐름을 나누어 관리하거나, 긴 파이프라인을 단계별로 쪼갤 때 유용해요.

## Workflow

### 언제 써요

- 여러 워크플로우에서 **공통 로직을 재사용**하고 싶을 때 써요.
- 긴 흐름을 **모듈 단위로 분리**해 가독성을 높이고 싶을 때 써요.
- 비동기로 다른 워크플로우를 **발화만** 하고 메인 흐름은 이어서 진행시키고 싶을 때 써요.

### 포트

입력 `in` 1개, 출력 `out` 1개예요. Sync 모드에서는 서브 워크플로우의 최종 출력이, Async 모드에서는 실행 요청 정보(executionId 등)가 나가요.

### Sync 모드 동작

1. `inputMapping`으로 입력을 구성해요. 미지정 시 부모 입력을 그대로 넘겨요.
2. 대상 워크플로우의 노드/연결선을 로드해 **부모 Execution 안에서 인라인으로 실행**해요.
3. 서브 워크플로우 노드들은 부모의 `executionId`, 노드 결과 캐시, 노드 맵을 공유해요.
4. 실행 히스토리에서 서브 워크플로우 노드가 부모 노드 바로 뒤에 이어서 보여요.
5. 서브 워크플로우의 최종 출력이 `output.result` 로 1단 래핑되어 `out` 포트로 나가요. 다운스트림은 `{{ $node["서브 워크플로우"].output.result.<field> }}` 로 접근해요.
6. 서브 워크플로우의 trigger 노드는 자동으로 스킵되고, 입력 데이터를 그대로 통과시켜요.

### 예시

<Example title="Sync 모드 — 공통 파이프라인 재사용">
```ts
// config
{
  "workflowId": "b6c7…-…-…",
  "workflowName": "Data Pipeline",
  "mode": "sync",
  "inputMapping": [
    { "paramName": "userId", "expression": "{{ $input.user.id }}" },
    { "paramName": "event",  "expression": "{{ $input.type }}" }
  ],
  "timeout": 300
}
```
</Example>

<Example title="Async 모드 — 알림 발화">
```ts
// config
{
  "workflowId": "{{ $var.notifyWorkflowId }}",
  "mode": "async",
  "inputMapping": [
    { "paramName": "channel", "expression": "{{ $input.channel }}" }
  ]
}

// out 포트 출력
{
  "executionId": "9f2c-…-…",
  "workflowId": "…",
  "status": "started"
}

// 표현식 접근 예
// $node["X"].output.executionId  → "9f2c-…-…"
// $node["X"].status              → "started"  (top-level status)
```
</Example>

<Callout type="warn">
대상 워크플로우가 삭제되거나 비활성화되면 설정 화면에 경고가 표시돼요. 캔버스 요약에도 `⚠ Missing workflow` 배지가 나타나요.
</Callout>

<Callout type="tip">
드롭다운은 같은 워크스페이스에 속한 워크플로우만 보여주고, 편집 중인 워크플로우 자신은 목록에서 제외돼요. 비활성 워크플로우는 `(inactive)` 라벨이 붙어요.
</Callout>

## 캔버스 요약

`{workflowName 또는 workflowId} · {mode}` 형식으로 표시돼요. 이름이 있으면 이름이 보이고, 없으면 ID가 보여요. 예: `Data Pipeline · sync`.

## 팁 & 참고

- 입력 매핑 없이 호출하면 부모 입력이 통째로 서브 워크플로우로 넘어가요. 필요한 파라미터만 명시적으로 넘기고 싶다면 `inputMapping`을 사용해요.
- 서브 워크플로우의 실행 결과와 로그는 [실행 결과](/docs/05-run-and-debug/run-results)에서 함께 확인할 수 있어요.
- 파워유저 팁: Sync 모드에서는 서브 워크플로우가 같은 실행 안에서 돌아가요. 덕분에 부모 쪽에서 `{{ $node["서브 워크플로우"].output.result.<field> }}` 처럼 서브 결과를 바로 참조할 수 있고, 실행 타임라인도 한 줄로 이어져 디버깅이 편해요.
