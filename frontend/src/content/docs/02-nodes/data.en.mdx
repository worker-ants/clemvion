This page covers the two data nodes. **Transform** chains visual transformation operations; **Code** lets you reshape data freely with sandboxed JavaScript.

## Transform

### When to use it

- When you want to rename fields, set/remove values, or convert types **without writing code**.
- Great for quick string/number/date ops, array filter/sort, and object pick/omit.

### Example

<Example title="Rename a field and derive a new one">
```ts
// operations
[
  { type: "rename_field", params: { from: "user.name", to: "userName" } },
  { type: "set_field",    params: { field: "fullName", value: "{{ $input.first + ' ' + $input.last }}" } },
  { type: "remove_field", params: { field: "tempData" } }
]
```
</Example>

<Callout type="note">
All `field` parameters support dot/bracket nested paths (e.g. `user.profile.name`, `items[0].id`). If the path doesn't exist or the type doesn't match, the op leaves the value untouched and continues to the next one.
</Callout>

## Code

### When to use it

- For **complex logic** that Transform can't express cleanly.
- Inline utility calculations, custom aggregation, conditional restructuring.

### Fields

### Example

<Example title="Pick active items and reshape them">
```ts
// code
const items = $input.data;

const result = items
  .filter((i) => i.active)
  .map((i) => ({
    id: i.id,
    name: i.name.trim(),
    score: i.score * 100,
  }));

console.log("filtered", result.length);

return { items: result };
```
</Example>

<Callout type="warn">
You can mutate `$vars` freely during code execution, but the real workflow variable store is updated **by full replacement only after the code finishes successfully**. If the code throws, the previous state is preserved.
</Callout>

## Canvas summary

- Transform: `{N} operations` (e.g. `3 operations`)
- Code: `{language} · {N} lines` (e.g. `JavaScript · 12 lines`)

## Tips & notes

- Use Transform for simple reshaping and Code for complex logic — keeping them separated keeps things maintainable.
- The division of labor between expressions and Code is discussed in [Expression basics](/docs/03-expression-language/basics) and the [cheatsheet](/docs/03-expression-language/cheatsheet).
- Power-user tip: the Code node runs in a sandbox that blocks `require`, file system, and network access. The default timeout is 30 seconds, configurable in node settings. For external API calls, split them out into an HTTP Request node.
