이 페이지에서는 데이터 노드 두 종류를 설명해요. **Transform**은 시각적 빌더로 변환 연산을 체이닝하고, **Code**는 샌드박스 환경의 JavaScript로 자유롭게 가공해요.

## Transform

### 언제 써요

- **코딩 없이** 필드 이름을 바꾸거나, 값을 설정·제거하거나, 타입을 변환하고 싶을 때 써요.
- 간단한 문자열·숫자·날짜 연산, 배열 필터/정렬, 객체 pick/omit이 필요할 때 유용해요.

### 예시

<Example title="필드 이름 바꾸고 파생 필드 만들기">
```ts
// operations
[
  { type: "rename_field", params: { from: "user.name", to: "userName" } },
  { type: "set_field",    params: { field: "fullName", value: "{{ $input.first + ' ' + $input.last }}" } },
  { type: "remove_field", params: { field: "tempData" } }
]
```
</Example>

<Callout type="note">
모든 `field` 파라미터는 dot/bracket 중첩 경로를 지원해요(예: `user.profile.name`, `items[0].id`). 경로가 없거나 타입이 맞지 않으면 해당 연산은 원값을 유지하고 다음 연산으로 넘어가요.
</Callout>

## Code

### 언제 써요

- Transform으로 표현하기 어려운 **복잡한 로직**이 필요할 때 써요.
- 작은 유틸 계산, 커스텀 집계, 조건부 재구조화를 인라인으로 처리할 때 유용해요.

### 예시

<Example title="활성 항목만 뽑아 재구조화">
```ts
// code
const items = $input.data;

const result = items
  .filter((i) => i.active)
  .map((i) => ({
    id: i.id,
    name: i.name.trim(),
    score: i.score * 100,
  }));

console.log("filtered", result.length);

return { items: result };
```
</Example>

## 캔버스 요약

- Transform: `{N} operations` (예: `3 operations`)
- Code: `{language} · {N} lines` (예: `JavaScript · 12 lines`)

## 팁 & 참고

- 간단한 변환은 Transform, 복잡한 로직은 Code로 나눠 쓰면 유지보수가 쉬워요.
- 표현식과 Code의 역할 구분은 [표현식 기본](/docs/04-expression-language/basics)과 [치트시트](/docs/04-expression-language/cheatsheet)를 참고해요.
- 파워유저 팁: Code 노드는 샌드박스에서 실행돼 `require`, 파일 시스템, 네트워크 호출이 차단돼요. 기본 타임아웃은 30초이고 노드 설정에서 조절할 수 있어요. 외부 API 호출이 필요하면 HTTP Request 노드로 분리해요.
