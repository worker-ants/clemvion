이 페이지에서는 노드가 어떤 요소로 구성되는지, 어떤 포트 규칙을 따르는지, 데이터와 에러가 어떻게 흘러가는지를 설명해요. 개별 노드의 상세 스펙은 카테고리별 페이지에서 이어서 다뤄요.

## 노드는 무엇이에요

노드는 워크플로우의 **단위 작업**이에요. 왼쪽에서 입력을 받아 설정대로 일한 뒤, 오른쪽으로 결과를 내보내요. 모든 노드는 동일한 공통 구조(타입·카테고리·입출력 포트·설정 스키마)를 따르기 때문에, 어떤 노드든 같은 방식으로 배치하고 연결할 수 있어요.

- **타입·카테고리**로 각 노드의 성격을 구분해요 (예: `if_else` → logic, `ai_agent` → ai).
- **포트**로 이어지는 데이터 흐름을 정의해요.
- **설정 스키마**로 폼의 필드와 검증 규칙이 결정돼요. 팔레트·설정 패널 UI는 이 스키마를 그대로 해석해서 자동으로 만들어져요.

## 포트 시스템

### 입력 포트와 출력 포트

- **입력 포트**는 노드 좌측, **출력 포트**는 우측에 배치돼요.
- 기본 식별자는 `in`, `out`이에요. 분기 노드는 `true`/`false`, `success`/`error` 처럼 역할별 ID를 써요.
- 동적으로 추가되는 포트(예: Switch의 케이스, Text Classifier의 카테고리, Presentation의 버튼 포트)는 **UUID v4**를 ID로 받아요. 이름을 바꾸거나 재정렬해도 ID가 바뀌지 않아서 **연결선이 그대로 유지돼요**.

## 캔버스 요약

노드 본체 3번째 줄에 설정 내용이 한 줄로 요약돼 표시돼요. 필수 설정이 비어 있으면 `⚠ Not configured`(앰버색) 배지가 붙어요. 줌이 50% 미만이면 요약이 숨겨져요.
