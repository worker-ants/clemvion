This page explains how to register the LLM providers used by AI Agent, Text Classifier, and Information Extractor, and how to set their default models and parameters. Knowledge Base embedding models also reuse the providers defined here.

## Supported providers

The LLM client abstraction layer normalizes provider differences, so AI nodes call them uniformly once the config is correct.

## Adding a provider

Click `[+ Add Provider]` at the top-right of `/llm-configs`. Required fields vary slightly by provider.

<Steps>
<Step>Pick a provider type from the dropdown: OpenAI · Anthropic · Google AI · Azure OpenAI · Local.</Step>
<Step>Enter an alias and the API Key. For Azure / Local (custom endpoints), also enter a `Base URL`.</Step>
<Step>Click `Load models` to fetch the provider's model list and pick from the dropdown, or type a default model ID directly (e.g. `gpt-4o`, `claude-sonnet-4-6`, `llama3`). The form can fetch the list before saving using the credentials you entered.</Step>
<Step>Click `Test Connection` to verify a lightweight call (e.g. list-models) via `POST /api/llm-configs/:id/test`.</Step>
<Step>Save. Mark your most-used provider as `⭐ Default` so newly-created AI nodes auto-select it.</Step>
</Steps>

<Callout type="warn">API Keys are encrypted with AES-256-GCM. API responses return only masked previews like `sk-...xxxx`, so keep the original key elsewhere (usually the provider console).</Callout>

Provider-level defaults are inherited as initial values by every AI node. Each node can override them independently.

<Example title="Local (Ollama) provider example">
```ts
// POST /api/llm-configs request body
{
  "provider": "local",
  "name": "Ollama (on prem)",
  "baseUrl": "http://localhost:11434",
  "apiKey": "",                      // empty string if not needed
  "defaultModel": "llama3",
  "defaultParams": {
    "temperature": 0.5,
    "maxTokens": 2048
  },
  "isDefault": false
}
```
</Example>

## Tips & notes

- Change the default provider with a single call to `PATCH /api/llm-configs/:id/set-default`. Existing AI nodes aren't affected — only newly created ones.
- It's fine to leave a provider without embedding support as the default. Knowledge Bases pick the embedding model per collection, so there's no conflict. See [Knowledge Base](/docs/05-integrations-and-config/knowledge-base).
- How to tune per-node parameters: [AI nodes](/docs/02-nodes/ai).
- For saved configs, list models via `GET /api/llm-configs/:id/models`. Before saving (while still in the form), use `POST /api/llm-configs/preview-models` with `{ provider, apiKey, baseUrl? }` — the preview endpoint never persists the API key and only uses it for the current request.
- For Azure OpenAI, the model ID must be the deployment name. Using the standard model ID returns 404.
